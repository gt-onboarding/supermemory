Überwachen Sie Ihre Supermemory-Nutzung mit detaillierten Analysen zu API-Aufrufen, Fehlern und Leistungskennzahlen.

<div id="overview">
  ## Übersicht
</div>

Die Analytics-API liefert umfassende Einblicke in Ihre Supermemory-Nutzung:

- **Nutzungsstatistiken**: Verfolgen Sie API-Aufrufe nach Typ, stündliche Trends und eine Aufschlüsselung nach API-Key
- **Fehlerüberwachung**: Ermitteln Sie die häufigsten Fehlertypen und Muster
- **Detaillierte Protokolle**: Greifen Sie für Debugging auf vollständige Request-/Response-Protokolle zu
- **Leistungskennzahlen**: Überwachen Sie durchschnittliche Antwortzeiten und Verarbeitungsdauer

<Note>
Analytics-Daten sind für Ihre gesamte Organisation verfügbar und können nach Zeitraum gefiltert werden.
</Note>

<div id="usage-statistics">
  ## Nutzungsstatistiken
</div>

Erhalten Sie umfassende Nutzungsstatistiken mit stündlichen Aufschlüsselungen und Kennzahlen pro Schlüssel.

<div id="endpoint">
  ### Endpoint
</div>

`GET /v3/analytics/usage`

<div id="parameters">
  ### Parameter
</div>

| Parameter | Typ | Beschreibung |
|-----------|------|-------------|
| `from` | string (ISO 8601) | Startdatum/-uhrzeit des Zeitraums |
| `to` | string (ISO 8601) | Enddatum/-uhrzeit des Zeitraums |
| `period` | string | Alternative zu `from`: `1h`, `24h`, `7d`, `30d` |
| `page` | integer | Seitennummer für die Paginierung (Standard: 1) |
| `limit` | integer | Einträge pro Seite (Standard: 20, maximal: 100) |

<div id="example-request">
  ### Beispielanfrage
</div>

<CodeGroup>

```typescript TypeScript
// Nutzung der letzten 24 Stunden abrufen
const usage = await fetch('https://api.supermemory.ai/v3/analytics/usage?period=24h', {
  headers: {
    'Authorization': `Bearer ${SUPERMEMORY_API_KEY}`
  }
});

const data = await usage.json();
```

```python Python
import requests
from datetime import datetime, timedelta

# Nutzung der letzten 7 Tage abrufen
response = requests.get(
    'https://api.supermemory.ai/v3/analytics/usage',
    params={'period': '7d'},
    headers={'Authorization': f'Bearer {SUPERMEMORY_API_KEY}'}
)

data = response.json()
```

```bash cURL
# Nutzung für einen bestimmten Datumsbereich abrufen
curl -X GET "https://api.supermemory.ai/v3/analytics/usage?from=2024-01-01T00:00:00Z&to=2024-01-31T23:59:59Z" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"
```

</CodeGroup>

<div id="error-monitoring">
  ## Fehlerüberwachung
</div>

Erfassen und analysieren Sie Fehler, um Probleme zu erkennen und die Zuverlässigkeit zu verbessern.

<div id="endpoint">
  ### Endpunkt
</div>

`GET /v3/analytics/errors`

<div id="parameters">
  ### Parameter
</div>

Entspricht dem Usage-Endpunkt – unterstützt `from`, `to`, `period`, `page` und `limit`.

<div id="example-request">
  ### Beispielanfrage
</div>

<CodeGroup>

```typescript TypeScript
// Fehler der letzten 24 Stunden abrufen
const errors = await fetch('https://api.supermemory.ai/v3/analytics/errors?period=24h', {
  headers: {
    'Authorization': `Bearer ${SUPERMEMORY_API_KEY}`
  }
});

const data = await errors.json();
```

```python Python
# Fehler überwachen und bei Spitzenwerten alarmieren
response = requests.get(
    'https://api.supermemory.ai/v3/analytics/errors?period=1h',
    headers={'Authorization': f'Bearer {SUPERMEMORY_API_KEY}'}
)

data = response.json()
```

```bash cURL
# Fehler der letzten 7 Tage abrufen
curl -X GET "https://api.supermemory.ai/v3/analytics/errors?period=7d" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"
```

</CodeGroup>

<div id="detailed-logs">
  ## Detaillierte Protokolle
</div>

Greifen Sie für Debugging und Auditing auf vollständige Anfragen-/Antwortprotokolle zu.

<div id="endpoint">
  ### Endpunkt
</div>

`GET /v3/analytics/logs`

<div id="parameters">
  ### Parameter
</div>

Wie beim Usage-Endpoint, plus optionale Filters:

- `type`: Nach Anfragetyp filtern (add, search, update, delete)
- `statusCode`: Nach HTTP-Statuscode filtern
- `keyId`: Nach spezifischem API-Key filtern

<div id="example-request">
  ### Beispielanfrage
</div>

<CodeGroup>

```typescript TypeScript
// Neueste fehlgeschlagene Anfragen abrufen
const logs = await fetch('https://api.supermemory.ai/v3/analytics/logs?period=1h&statusCode=500', {
  headers: {
    'Authorization': `Bearer ${SUPERMEMORY_API_KEY}`
  }
});

const data = await logs.json();
```

```python Python
# Nutzung eines bestimmten API-Keys debuggen
response = requests.get(
    'https://api.supermemory.ai/v3/analytics/logs',
    params={
        'keyId': 'key_abc123',
        'period': '24h'
    },
    headers={'Authorization': f'Bearer {SUPERMEMORY_API_KEY}'}
)

logs = response.json()['logs']

```

```bash cURL
# Alle Logs zum Debuggen abrufen
curl -X GET "https://api.supermemory.ai/v3/analytics/logs?period=1h&limit=50" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"
```

</CodeGroup>

<div id="rate-limits">
  ## Rate Limits
</div>

Analytics-Endpunkte unterliegen folgenden Rate Limits:

- 100 Anfragen pro Minute und Organisation
- Maximaler Zeitraum: 90 Tage
- Maximale Seitengröße: 100 Einträge

<Warning>
Analytics-Daten werden 90 Tage lang vorgehalten. Für eine längere Aufbewahrung exportieren Sie die Daten und speichern Sie sie in Ihren eigenen Systemen.
</Warning>
