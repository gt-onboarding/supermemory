Verbinde Google Drive, um documents mit OAuth-Authentifizierung und Unterstützung für benutzerdefinierte Apps in deine Supermemory-Wissensbasis zu synchronisieren.

<div id="quick-setup">
  ## Schnellstart
</div>

<div id="1-create-google-drive-connection">
  ### 1. Google-Drive-Verknüpfung erstellen
</div>

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    import Supermemory from 'supermemory';

    const client = new Supermemory({
      apiKey: process.env.SUPERMEMORY_API_KEY!
    });

    const connection = await client.connections.create('google-drive', {
      redirectUrl: 'https://yourapp.com/auth/google-drive/callback',
      containerTags: ['user-123', 'gdrive-sync'],
      documentLimit: 3000,
      metadata: {
        source: 'google-drive',
        department: 'engineering'
      }
    });

    // Benutzer zu Google OAuth weiterleiten
    window.location.href = connection.authLink;
    console.log('Authentifizierung läuft ab in:', connection.expiresIn);
    ```
  </Tab>
  <Tab title="Python">
    ```python
    from supermemory import Supermemory
    import os

    client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

    connection = client.connections.create(
        'google-drive',
        redirect_url='https://yourapp.com/auth/google-drive/callback',
        container_tags=['user-123', 'gdrive-sync'],
        document_limit=3000,
        metadata={
            'source': 'google-drive',
            'department': 'engineering'
        }
    )

    # Benutzer zu Google OAuth weiterleiten
    print(f'Weiterleiten zu: {connection.auth_link}')
    print(f'Läuft ab in: {connection.expires_in}')
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/connections/google-drive" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "redirectUrl": "https://yourapp.com/auth/google-drive/callback",
        "containerTags": ["user-123", "gdrive-sync"],
        "documentLimit": 3000,
        "metadata": {
          "source": "google-drive",
          "department": "engineering"
        }
      }'
    ```
  </Tab>
</Tabs>

<div id="2-handle-oauth-callback">
  ### 2. OAuth-Callback verarbeiten
</div>

Nachdem der Nutzer die Berechtigungen erteilt hat, leitet Google zu Ihrer Callback-URL weiter. Das verknüpfte Konto wird automatisch hergestellt.

<div id="delete-connection">
  ### Verbindung löschen
</div>

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    // Nach Verbindungs-ID löschen
    const result = await client.connections.deleteByID('connection_id_123');
    console.log('Verknüpftes Konto gelöscht:', result.id);

    // Nach provider und Container-Tags löschen
    const providerResult = await client.connections.deleteByProvider('google-drive', {
      containerTags: ['user-123']
    });
    console.log('Verknüpftes Konto für Provider gelöscht:', providerResult.id);
    ```
  </Tab>
  <Tab title="Python">
    ```python
    # Nach Verbindungs-ID löschen
    result = client.connections.delete_by_id('connection_id_123')
    print(f'Verknüpftes Konto gelöscht: {result.id}')

    # Nach provider und Container-Tags löschen
    provider_result = client.connections.delete_by_provider(
        'google-drive',
        container_tags=['user-123']
    )
    print(f'Verknüpftes Konto für Provider gelöscht: {provider_result.id}')
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    # Nach Verbindungs-ID löschen
    curl -X DELETE "https://api.supermemory.ai/v3/connections/connection_id_123" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY"

    # Response: {"id": "connection_id_123", "provider": "google-drive"}

    # Nach provider und Container-Tags löschen
    curl -X DELETE "https://api.supermemory.ai/v3/connections/google-drive" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "containerTags": ["user-123"]
      }'

    # Response: {"id": "conn_gd123", "provider": "google-drive"}
    ```
  </Tab>
</Tabs>

<Note>
Das Löschen eines verknüpften Kontos bewirkt:
- Beendet alle zukünftigen Synchronisierungen von Google Drive
- Entfernt die OAuth-Autorisierung
- Behält die bereits synchronisierten documents in Supermemory (sie werden nicht gelöscht)
</Note>

        'google-drive',

<div id="custom-oauth-application">
  ### Eigene OAuth-Anwendung
</div>

Konfigurieren Sie Ihre eigene Google-OAuth-App über die Settings-API:

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    // Organisations­einstellungen mit Ihrer Google-OAuth-App aktualisieren
    await client.settings.update({
      googleDriveCustomKeyEnabled: true,
      googleDriveClientId: 'your-google-client-id.googleusercontent.com',
      googleDriveClientSecret: 'your-google-client-secret'
    });

    // Aktuelle Einstellungen abrufen
    const settings = await client.settings.get();
    console.log('Google Drive Custom Key aktiviert:', settings.googleDriveCustomKeyEnabled);
    console.log('Client-ID konfiguriert:', !!settings.googleDriveClientId);
    ```
  </Tab>
  <Tab title="Python">
    ```python
    # Organisationseinstellungen mit Ihrer Google-OAuth-App aktualisieren
    client.settings.update(
        google_drive_custom_key_enabled=True,
        google_drive_client_id='your-google-client-id.googleusercontent.com',
        google_drive_client_secret='your-google-client-secret'
    )

    # Aktuelle Einstellungen abrufen
    settings = client.settings.get()
    print(f'Google Drive Custom Key aktiviert: {settings.google_drive_custom_key_enabled}')
    print(f'Client-ID konfiguriert: {bool(settings.google_drive_client_id)}')
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    # Organisationseinstellungen aktualisieren
    curl -X PATCH "https://api.supermemory.ai/v3/settings" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "googleDriveCustomKeyEnabled": true,
        "googleDriveClientId": "your-google-client-id.googleusercontent.com",
        "googleDriveClientSecret": "your-google-client-secret"
      }'

    # Aktuelle Einstellungen abrufen
    curl -X GET "https://api.supermemory.ai/v3/settings" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY"
    ```
  </Tab>
</Tabs>

<div id="document-filtering">
  ### Dokumentenfilterung
</div>

Konfigurieren Sie die Filterung über die Settings-API:

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    await client.settings.update({
      shouldLLMFilter: true,
      filterPrompt: "Only sync important business documents",
      includeItems: {
        // Ihre Include-Muster
      },
      excludeItems: {
        // Ihre Exclude-Muster
      }
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    client.settings.update(
        should_llm_filter=True,
        filter_prompt="Only sync important business documents",
        include_items={
            # Ihre Include-Muster
        },
        exclude_items={
            # Ihre Exclude-Muster
        }
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    # Dokumentenfilterung konfigurieren
    curl -X PATCH "https://api.supermemory.ai/v3/settings" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "shouldLLMFilter": true,
        "filterPrompt": "Only sync important business documents",
        "includeItems": {
          "patterns": ["*.pdf", "*.docx"],
          "folders": ["Important Documents", "Projects"]
        },
        "excludeItems": {
          "patterns": ["*.tmp", "*.backup"],
          "folders": ["Archive", "Trash"]
        }
      }'

    # Antwort: {
    #   "shouldLLMFilter": true,
    #   "filterPrompt": "Only sync important business documents",
    #   "includeItems": {...},
    #   "excludeItems": {...}
    # }
    ```
  </Tab>
</Tabs>

<Warning>
**Wichtige Hinweise:**
- OAuth-Tokens können ablaufen – prüfen Sie das Feld `expiresAt`
- Die Verarbeitung von Dokumenten erfolgt asynchron
- Verwenden Sie Container-Tags konsequent für das Filtern
- Überwachen Sie den Dokumentstatus, um fehlgeschlagene Synchronisierungen zu erkennen
</Warning>
