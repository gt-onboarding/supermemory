---
title: "Verknüpfte Konten erstellen"
description: "Erstellen Sie ein verknüpftes Konto, um Ihre Inhalte mit Supermemory zu synchronisieren"
---

Um ein verknüpftes Konto zu erstellen, senden Sie einfach eine `POST`-Anfrage an `/v3/connections/{provider}`.

<CodeGroup>
```typescript Typescript
import Supermemory from 'supermemory';

const client = new Supermemory({
  apiKey: process.env['SUPERMEMORY_API_KEY'], // Dies ist der Standard und kann weggelassen werden
});

const connection = await client.connections.create('notion');

console.debug(connection.authLink);
```

```python Python
import requests

url = "https://api.supermemory.ai/v3/connections/{provider}"

payload = {
    "redirectUrl": "<string>",
    "containerTags": ["<string>"],
    "metadata": {},
    "documentLimit": 5000
}
headers = {
    "Authorization": "Bearer <token>",
    "Content-Type": "application/json"
}

response = requests.request("POST", url, json=payload, headers=headers)

print(response.text)
```

```bash cURL
curl --request POST \
  --url https://api.supermemory.ai/v3/connections/{provider} \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
  "redirectUrl": "<string>",
  "containerTags": [
    "<string>"
  ],
  "metadata": {},
  "documentLimit": 5000
}'
```
</CodeGroup>

<div id="parameters">
  ### Parameter
</div>

- `provider`: Der provider, zu dem eine Verbindung hergestellt wird. Derzeit unterstützte provider sind `notion`, `google-drive`, `one-drive`
- `redirectUrl`: Die URL, zu der nach dem Erstellen des verknüpften Kontos weitergeleitet wird (Ihre App-URL)
- `containerTags`: Optional. Zum Partitionieren von Nutzer:innen, Organisationen usw. in Ihrer App.
    - Beispiel: `["user_123", "project_alpha"]`
- `metadata`: Optional. Beliebige Metadata, die Sie dem verknüpften Konto zuordnen möchten.
    - Diese Metadata wird jedem document hinzugefügt, das von diesem verknüpften Konto synchronisiert wird.
- `documentLimit`: Optional. Die maximale Anzahl an documents, die von diesem verknüpften Konto synchronisiert werden.
    - Standard: 10.000
    - Dies kann verwendet werden, um Kosten zu begrenzen und eine festgelegte Anzahl von documents für eine:n bestimmte:n Nutzer:in zu synchronisieren.

<div id="response">
  ## Antwort
</div>

supermemory sendet eine Antwort mit folgendem Schema:

```json
{
  "id": "<string>",
  "authLink": "<string>",
  "expiresIn": "<string>",
  "redirectsTo": "<string>"
}
```

Sie können den `authLink` verwenden, um den Benutzer zur Anmeldeseite des provider weiterzuleiten.
