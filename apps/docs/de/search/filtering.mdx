Supermemory unterstützt das Filtern von Suchergebnissen mittels Container-Tags, Metadata-Bedingungen und erweiterten Filtertechniken für die Endpunkte `/v3/search` und `/v4/search`.

<div id="container-tags">
  ## Container-Tags
</div>

Container-Tags gruppieren Speicher-Einträge nach Benutzer, Projekt oder Organisation. Sie sind die primäre Methode, um Suchergebnisse zu isolieren.

**Wichtig**: Container-Tags verwenden **exaktes Array-Matching**. Ein Dokument mit `["technology", "quantum-computing"]` passt NICHT zu einer Suche nach `["technology"]`. Die Arrays müssen identisch sein.

<div id="document-search-v3search">
  ### Dokumentensuche (v3/search)
</div>

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    // Einzelnes Container-Tag
    const results = await client.search.documents({
      q: "machine learning",
      containerTags: ["user_123"],
      limit: 10
    });

    // Mehrere Container-Tags
    const results2 = await client.search.documents({
      q: "project status",
      containerTags: ["project_ai", "team_research"],
      limit: 10
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    # Einzelnes Container-Tag
    results = client.search.documents(
        q="machine learning",
        container_tags=["user_123"],
        limit=10
    )

    # Mehrere Container-Tags
    results2 = client.search.documents(
        q="project status",
        container_tags=["project_ai", "team_research"],
        limit=10
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    # Einzelnes Container-Tag
    curl -X POST "https://api.supermemory.ai/v3/search" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "q": "machine learning",
        "containerTags": ["user_123"],
        "limit": 10
      }'
    ```
  </Tab>
</Tabs>

<Note>
**Unterschiede bei Container-Tags**:
- `/v3/search` verwendet `containerTags` (Plural-Array) mit exakter Array-Übereinstimmung
- `/v4/search` verwendet `containerTag` (singulärer String) für das Filtern mit einem einzelnen Tag
- Exakte Übereinstimmung bedeutet `["user", "project"]` ≠ `["user"]`
</Note>

<div id="filter-best-practices">
  ## Best Practices für Filter
</div>

<Callout type="info">
**Performance-Tipps**:
- Verwenden Sie Container-Tags als primären Filter (am schnellsten)
- Kombinieren Sie sie mit einfachen Metadata-Filtern für höhere Präzision
- Vermeiden Sie komplexe, verschachtelte OR-Bedingungen für bessere Performance
- Verwenden Sie numerische Operatoren nur bei Bedarf
</Callout>

<Warning>
**Filterstruktur**: Alle Bedingungen müssen in `AND`- oder `OR`-Arrays eingebettet sein. Auch einzelne Bedingungen benötigen den Array-Wrapper: `{"AND": [{"key": "category", "value": "ai"}]}`.
</Warning>

<Note>
**Unterstützte Operatoren**:
- **String**: Nur exakte Übereinstimmung
- **Numerisch**: `<=`, `>=`, `<`, `>`, `=`
- **Array**: `array_contains`, um zu prüfen, ob das Array den Wert enthält
- **Negation**: Funktioniert mit allen Filtertypen
</Note>
