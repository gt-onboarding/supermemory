Die Documents-Suche (`POST /v3/search`) bietet maximale Kontrolle über das Suchverhalten mit umfangreichen Parametern zur Feinabstimmung der Ergebnisse.

<div id="basic-implementation">
  ## Basisimplementierung
</div>

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    import Supermemory from 'supermemory';

    const client = new Supermemory({
      apiKey: process.env.SUPERMEMORY_API_KEY!
    });

    const results = await client.search.documents({
      q: "machine learning neural networks",
      limit: 5
    });

    console.log(`Found ${results.total} documents in ${results.timing}ms`);

    // Beispielausgabe-Struktur
    results.results.forEach((doc, i) => {
      console.log(`${i + 1}. ${doc.title} (Relevanzwert: ${doc.score})`);
      console.log(`   ${doc.chunks.length} Chunks gefunden`);
    });
    ```
  </Tab>

  <Tab title="Python">
    ```python
    from supermemory import Supermemory
    import os

    client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

    results = client.search.documents(
        q="machine learning neural networks",
        limit=5
    )

    print(f"Found {results.total} documents in {results.timing}ms")

    # Beispielausgabe-Struktur
    for i, doc in enumerate(results.results):
        print(f"{i + 1}. {doc.title} (Relevanzwert: {doc.score})")
        print(f"   {len(doc.chunks)} Chunks gefunden")
    ```
  </Tab>


<div id="container-tags-filtering">
  ## Filtern mit Container-Tags
</div>

Container-Tags sind die wichtigste Methode, um Suchergebnisse nach Benutzer, Projekt oder Organisation abzugrenzen.

**Wesentliche Eigenschaften:**

- **Array-basiert**: Im Unterschied zu `/v4/search` akzeptiert dieser Endpoint mehrere Container-Tags als Array
- **Exakte Array-Übereinstimmung**: documents müssen genau dasselbe Container-Tags-Array besitzen, um übereinzustimmen

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const results = await client.search.documents({
      q: "quarterly reports",
      containerTags: ["user_123"],
      limit: 10
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    results = client.search.documents(
        q="quarterly reports",
        container_tags=["user_123"],
        limit=10
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/search" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "q": "quarterly reports",
        "containerTags": ["user_123"],
        "limit": 10
      }'
    ```
  </Tab>
</Tabs>

<div id="metadata-filtering">
  ## Metadata-Filterung
</div>

Metadata-Filterung ermöglicht komplexe Bedingungen für strukturierte Daten, die an Ihre documents angehängt sind. Im Backend wird dafür eine SQL-ähnliche Abfragekonstruktion verwendet, die explizite AND/OR-Strukturen erfordert.

**Regeln für die Filterstruktur:**

* **Bedingungen müssen** in AND- oder OR-Arrays gekapselt sein, auch bei einzelnen Bedingungen
* **Unterstützt String-Abgleich** (exakt), numerische Operatoren und Array-Contains
* **Beliebige Bedingung negieren** mit `negate: true`
* **Kombinierbar mit Container-Tags** – beide filters werden angewendet

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const results = await client.search.documents({
      q: "machine learning",
      filters: {
        AND: [
          {
            key: "category",
            value: "technology",
            negate: false
          },
          {
            filterType: "numeric",
            key: "readingTime",
            value: "5",
            negate: false,
            numericOperator: "<="
          }
        ]
      },
      limit: 10
    });
    ```
  </Tab>

  <Tab title="Python">
    ```python
    results = client.search.documents(
        q="machine learning",
        filters={
            "AND": [
                {
                    "key": "category",
                    "value": "technology",
                    "negate": False
                },
                {
                    "filterType": "numeric",
                    "key": "readingTime",
                    "value": "5",
                    "negate": False,
                    "numericOperator": "<="
                }
            ]
        },
        limit=10
    )
    ```
  </Tab>

  <Tab title="cURL">

<div id="array-contains-filtering">
  ## Filtern mit array_contains
</div>

Wenn Ihre Metadata Arrays enthalten (z. B. Teilnehmerlisten, Tags oder Kategorien), verwenden Sie `array_contains`, um zu prüfen, ob das Array einen bestimmten Wert enthält.

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const results = await client.search.documents({
      q: "meeting discussion",
      filters: {
        AND: [
          {
            key: "participants",
            value: "john.doe",
            filterType: "array_contains"
          }
        ]
      },
      limit: 5
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    results = client.search.documents(
        q="meeting discussion",
        filters={
            "AND": [
                {
                    "key": "participants",
                    "value": "john.doe",
                    "filterType": "array_contains"
                }
            ]
        },
        limit=5
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/search" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -D '{
        "q": "meeting discussion",

          ]
        },
        "limit": 5
      }'
    ```

</Tabs>

<div id="threshold-control">

</div>

<Tabs>

<div id="query-rewriting">
  ## Query Rewriting
</div>

Verbessern Sie die Suchgenauigkeit mit automatischem Query Rewriting:

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const results = await client.search.documents({
      q: "What is the capital of France?",
      rewriteQuery: true,  // +400 ms Latenz, aber bessere Ergebnisse
      limit: 5
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    results = client.search.documents(
        q="What is the capital of France?",
        rewrite_query=True,  # +400 ms Latenz, aber bessere Ergebnisse
        limit=5
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/search" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "q": "What is the capital of France?",
        "rewriteQuery": true,
        "limit": 5
      }'
    ```
  </Tab>
</Tabs>

<Note>
Query Rewriting erzeugt mehrere Varianten der Suchanfrage, durchsucht alle und führt anschließend die Ergebnisse zusammen. Keine zusätzlichen Kosten, jedoch ~400 ms zusätzliche Latenz.
</Note>

<div id="full-context-options">

</div>

Vollständige Dokumentinhalte und Zusammenfassungen einschließen:

<Tabs>

    ```typescript
    const results = await client.search.documents({
      q: "research findings",
      includeFullDocs: true,    // Vollständigen Dokumentinhalt einschließen
    ```

    ```python
    results = client.search.documents(

        include_summary=True,      # Dokumentzusammenfassungen einschließen

        limit=5
    ```
