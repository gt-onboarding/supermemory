<div id="direct-updates">
  ## Direkte Updates
</div>

Aktualisieren Sie vorhandene Speicher über ihre ID, wenn Sie genau wissen, welchen Speicher Sie ändern möchten. Änderungen lösen eine erneute Verarbeitung durch die gesamte Pipeline aus.

<CodeGroup>

```typescript Typescript
import Supermemory from 'supermemory';

const client = new Supermemory({
  apiKey: process.env.SUPERMEMORY_API_KEY!
});

// Update per Speicher-ID
const updated = await client.memories.update('memory_id_123', {
  content: 'Updated content here',
  metadata: { version: 2, updated: true }
});

console.log(updated.status); // "queued" zur erneuten Verarbeitung
console.log(updated.id); // "memory_id_123"
```

```python Python
from supermemory import Supermemory
import os

client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

# Update per Speicher-ID
updated = client.memories.update(
    'memory_id_123',
    content='Updated content here',
    metadata={'version': 2, 'updated': True}
)

print(f"Status: {updated.status}")  # "queued" zur erneuten Verarbeitung
print(f"ID: {updated.id}")  # "memory_id_123"
```

```bash cURL
curl -X PATCH "https://api.supermemory.ai/v3/documents/memory_id_123" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Updated content here",
    "metadata": {"version": 2, "updated": true}
  }'
```

</CodeGroup>

<div id="upserts-using-customid">
  ## Upserts mit customId
</div>

Verwenden Sie `customId` für idempotente Operationen: Bei derselben `customId` in `add()` wird der vorhandene Speicher-Eintrag aktualisiert, statt Duplikate zu erzeugen.

<CodeGroup>

```typescript Typescript
import Supermemory from 'supermemory';

const client = new Supermemory({
  apiKey: process.env.SUPERMEMORY_API_KEY!
});

const customId = 'user-note-001';

// First call creates memory
const created = await client.memories.add({
  content: 'Initial content',
  customId: customId,
  metadata: { version: 1 }
});

console.log('Created memory:', created.id);

// Second call with same customId updates existing
const updated = await client.memories.add({
  content: 'Updated content',
  customId: customId,         // Same customId = upsert
  metadata: { version: 2 }
});
```

```python Python
from supermemory import Supermemory
import os

client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

custom_id = 'user-note-001'

# First call creates memory
created = client.memories.add(
    content='Initial content',
    custom_id=custom_id,
    metadata={'version': 1}
)

print(f'Created memory: {created.id}')

# Second call with same customId updates existing
updated = client.memories.add(
    content='Updated content',
    custom_id=custom_id,     # Same customId = upsert
    metadata={'version': 2}
)

print(f'Updated memory: {updated.id}')
print(f'Same memory? {created.id == updated.id}')  # True
```

```bash cURL
# First call - creates memory
curl -X POST "https://api.supermemory.ai/v3/documents" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Initial content",
    "customId": "user-note-001",
    "metadata": {"version": 1}
  }'

# Response: {"id": "mem_abc123", "status": "queued", "customId": "user-note-001"}

# Second call - updates existing (same customId)
curl -X POST "https://api.supermemory.ai/v3/documents" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Updated content",
    "customId": "user-note-001",
    "metadata": {"version": 2}
  }'

# Response: {"id": "mem_abc123", "status": "queued", "customId": "user-note-001"}
# Note: Same ID returned - memory was updated, not created
```

</CodeGroup>

<Note>
Die `customId` ermöglicht Idempotenz für alle Endpunkte. Die `memoryId` unterstützt keine Idempotenz – nur die `customId`.
</Note>

<div id="single-delete">
  ## Einzelnes Löschen
</div>

Löschen Sie einzelne Speicher-Einträge über ihre ID. Dies ist ein endgültiges Hard-Delete ohne Möglichkeit zur Wiederherstellung.

<CodeGroup>

```typescript Typescript
// Hard delete – entfernt den Speicher-Eintrag dauerhaft
await client.memories.delete('memory_id_123');
console.log('Memory deleted successfully');
```

```python Python
# Hard delete – entfernt den Speicher-Eintrag dauerhaft
client.memories.delete('memory_id_123')
print('Memory deleted successfully')

# Fehlerbehandlung für einzelnes Löschen
try:
    client.memories.delete('memory_id_123')
    print('Delete successful')
except NotFoundError:
    print('Memory not found or already deleted')
except AuthenticationError:
    print('Authentication failed')
except Exception as e:
    print(f'Delete failed: {e}')
```

```bash cURL
curl -X DELETE "https://api.supermemory.ai/v3/documents/memory_id_123" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"

# Response: 204 No Content (success)
# Response: 404 Not Found (memory doesn't exist)
```

</CodeGroup>

<div id="bulk-delete-by-ids">
  ## Massenlöschung über IDs
</div>

Löschen Sie mehrere Speicher-Einträge auf einmal, indem Sie ein Array von Speicher-IDs angeben. Maximal 100 IDs pro Anfrage.

<CodeGroup>

```typescript Typescript
// Massenlöschung über Speicher-IDs
const result = await client.memories.bulkDelete({
  ids: [
    'memory_id_1',
    'memory_id_2',
    'memory_id_3',
    'non_existent_id'  // Wird unter "errors" gemeldet
  ]
});

console.log('Ergebnis der Massenlöschung:', result);
// Output: {
//   success: true,
//   deletedCount: 3,
//   errors: [
//     { id: "non_existent_id", error: "Memory not found" }
//   ]
// }
```


<div id="soft-delete-implementation">
  ### Implementierung von Soft Delete
</div>

Für Anwendungen, die Audit-Trails oder Wiederherstellungsmechanismen benötigen, implementieren Sie Soft-Delete-Muster mithilfe von Metadata:

<CodeGroup>

```typescript Typescript
// Soft-Delete-Muster mit Metadata
await client.memories.update('memory_id', {
  metadata: {
    deleted: true,
    deletedAt: new Date().toISOString(),
    deletedBy: 'user_123'
  }
});

// Gelöschte Speicher-Einträge bei Suchanfragen ausschließen
const activeMemories = await client.memories.list({
  filters: JSON.stringify({
    AND: [
      { key: "deleted", value: "true", negate: true }
    ]
  })
});

console.log('Aktive Speicher-Einträge:', activeMemories.results.length);
```

```python Python
from datetime import datetime
import json

# Soft-Delete-Muster mit Metadata
client.memories.update('memory_id', {
    'metadata': {
        'deleted': True,
        'deletedAt': datetime.now().isoformat(),
        'deletedBy': 'user_123'
    }
})

# Gelöschte Speicher-Einträge ausschließen
active_memories = client.memories.list(
    filters=json.dumps({
        "AND": [
            {"key": "deleted", "value": "true", "negate": True}
        ]
    })
)

print(f'Aktive Speicher-Einträge: {len(active_memories.results)}')
```

```bash cURL
# Soft Delete mit Metadata
curl -X PATCH "https://api.supermemory.ai/v3/documents/memory_id" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "deleted": true,
      "deletedAt": "2024-01-15T10:30:00Z",
      "deletedBy": "user_123"
    }
  }'

# Antwort: {"id": "memory_id", "status": "queued"}
```

<div id="batch-processing-for-large-operations">
  ### Stapelverarbeitung für umfangreiche Vorgänge
</div>

<CodeGroup>

```typescript Typescript
// Große Mengen an Speicher-Einträgen sicher in Batches löschen
async function batchDeleteMemories(memoryIds: string[], batchSize = 100) {
  const results = [];

  for (let i = 0; i < memoryIds.length; i += batchSize) {
    const batch = memoryIds.slice(i, i + batchSize);

    console.log(`Verarbeite Batch ${Math.floor(i/batchSize) + 1} von ${Math.ceil(memoryIds.length/batchSize)}`);

    try {
      const result = await client.memories.bulkDelete({ ids: batch });
      results.push(result);

      // Kurze Pause zwischen Batches, um Rate Limits zu vermeiden
      if (i + batchSize < memoryIds.length) {
        await new Promise(resolve => setTimeout(resolve, 1000));
      }
    } catch (error) {
      console.error(`Batch ${Math.floor(i/batchSize) + 1} fehlgeschlagen:`, error);
      results.push({ success: false, error: error.message, batch });
    }
  }

  // Ergebnisse aggregieren
  const totalDeleted = results
    .filter(r => r.success)
    .reduce((sum, r) => sum + (r.deletedCount || 0), 0);

  console.log(`Insgesamt gelöscht: ${totalDeleted} von ${memoryIds.length}`);
  return { totalDeleted, results };
}
```

```python Python
import time
import math

def batch_delete_memories(memory_ids, batch_size=100):
    """Große Mengen an Speicher-Einträgen sicher in Batches löschen"""
    results = []

    for i in range(0, len(memory_ids), batch_size):
        batch = memory_ids[i:i + batch_size]
        batch_num = i // batch_size + 1
        total_batches = math.ceil(len(memory_ids) / batch_size)

        print(f'Verarbeite Batch {batch_num} von {total_batches}')

        try:
            result = client.memories.bulk_delete(ids=batch)
            results.append(result)

            # Kurze Pause zwischen Batches, um Rate Limits zu vermeiden
            if i + batch_size < len(memory_ids):
                time.sleep(1)
        except Exception as error:
            print(f'Batch {batch_num} fehlgeschlagen: {error}')
            results.append({'success': False, 'error': str(error), 'batch': batch})

    # Ergebnisse aggregieren
    total_deleted = sum(
        r.get('deletedCount', 0) for r in results if r.get('success')
    )

    print(f'Insgesamt gelöscht: {total_deleted} von {len(memory_ids)}')
    return {'totalDeleted': total_deleted, 'results': results}
```

```bash cURL
# Beispielskript für Stapelverarbeitung
#!/bin/bash

MEMORY_IDS=("id1" "id2" "id3")  # Ihr Array mit Speicher-IDs
BATCH_SIZE=100
TOTAL_DELETED=0

# In Batches verarbeiten
for ((i=0; i<${#MEMORY_IDS[@]}; i+=BATCH_SIZE)); do
    batch=("${MEMORY_IDS[@]:i:BATCH_SIZE}")
    batch_json=$(printf '%s\n' "${batch[@]}" | jq -R . | jq -s .)

    echo "Verarbeite Batch $((i/BATCH_SIZE + 1))"

    response=$(curl -s -X DELETE \
      "https://api.supermemory.ai/v3/documents/bulk" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d "{\"ids\": $batch_json}")

    deleted_count=$(echo "$response" | jq -r '.deletedCount // 0')
    TOTAL_DELETED=$((TOTAL_DELETED + deleted_count))

    echo "Batch gelöscht: $deleted_count Speicher-Einträge"
    sleep 1  # Schutz vor Rate Limits
done

echo "Insgesamt gelöscht: $TOTAL_DELETED Speicher-Einträge"
```

<div id="best-practices">
  ## Best Practices
</div>

<div id="update-operations">
  ### Update-Vorgänge
</div>

1. **customId für idempotente Updates verwenden** - Verhindert doppelte Speicher-Einträge und ermöglicht sichere Wiederholungen
2. **Verarbeitungsstatus überwachen** - Updates stoßen die vollständige Reprocessing-Pipeline an
3. **Metadata sorgfältig behandeln** - Updates ersetzen die angegebenen Metadata-Schlüssel
4. **Korrektes Error-Handling implementieren** - Ein Speicher kann zwischen Vorgängen gelöscht werden

<div id="delete-operations">
  ### Löschvorgänge
</div>

1. **Hard Delete ist endgültig** - Es gibt keinen Wiederherstellungsmechanismus
2. **Bulk-Operationen effizient nutzen** - Maximal 100 IDs pro Bulk-Delete-Anfrage
3. **Soft-Delete-Patterns erwägen** - Verwenden Sie Metadata-Flags für wiederherstellbares Löschen
4. **Große Vorgänge bündeln** - Vermeiden Sie Rate Limits durch korrektes Batching
5. **Anwendungszustand bereinigen** - Aktualisieren Sie Ihre UI bzw. Ihren Cache nach Löschvorgängen
