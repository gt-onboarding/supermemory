Memory Router एक पारदर्शी प्रॉक्सी है जो आपके एप्लिकेशन और आपके LLM provider के बीच रहता है और बिना किसी कोड बदलाव के संदर्भ और संग्रहीत मेमोरीज़ को स्वचालित रूप से प्रबंधित करता है।

<Note>
  **लाइव डेमो**: इसे क्रियान्वयन में देखने के लिए [supermemory.chat](https://supermemory.chat) पर Memory Router आज़माएँ।
</Note>

<div id="what-is-the-memory-router">
  ## Memory Router क्या है?
</div>

Memory Router आपके LLM अनुप्रयोगों को देता है:

* **असीमित संदर्भ**: अब token limit नहीं — बातचीत अनिश्चितकाल तक चल सकती है
* **स्वचालित memory प्रबंधन**: बुद्धिमानी से चंक करता है, स्टोर करता है, और प्रासंगिक संदर्भ रिट्रीव करता है
* **कोड में शून्य बदलाव**: आपके मौजूदा OpenAI-संगत क्लाइंट्स के साथ काम करता है
* **लागत अनुकूलन**: बुद्धिमान संदर्भ प्रबंधन के जरिए token लागत में 70% तक बचत

<div id="how-it-works">
  ## यह कैसे काम करता है
</div>

<Steps>
  <Step title="प्रॉक्सी रिक्वेस्ट">
    आपकी एप्लिकेशन सीधे आपके LLM provider को भेजने के बजाय Supermemory को रिक्वेस्ट भेजती है
  </Step>

  <Step title="संदर्भ प्रबंधन">
    Supermemory स्वचालित रूप से:

    * लंबी बातचीत से अनावश्यक संदर्भ हटाता है
    * पिछली इंटरैक्शंस से प्रासंगिक संग्रहीत मेमोरीज़ खोजता है
    * आपके प्रॉम्प्ट में सबसे प्रासंगिक संदर्भ जोड़ता है
  </Step>

  <Step title="LLM को फ़ॉरवर्ड">
    ऑप्टिमाइज़्ड रिक्वेस्ट को आपके चुने हुए LLM provider को फ़ॉरवर्ड किया जाता है
  </Step>

  <Step title="असिंक्रोनस मेमोरी निर्माण">
    नई संग्रहीत मेमोरीज़ प्रतिक्रिया को ब्लॉक किए बिना असिंक्रोनस रूप से बनाई जाती हैं
  </Step>
</Steps>

<div id="key-benefits">
  ## मुख्य लाभ
</div>

<div id="for-developers">
  ### डेवलपर्स के लिए
</div>

* **ड्रॉप-इन इंटीग्रेशन**: बस अपना बेस URL बदलें — किसी और कोड बदलाव की जरूरत नहीं
* **प्रोवाइडर-अज्ञेय**: OpenAI, Anthropic, Google, Groq और अन्य के साथ काम करता है
* **शेयर्ड मेमोरी पूल**: API के जरिए बनाई गई संग्रहीत मेमोरीज़ Router में उपलब्ध होती हैं, और इसका उल्टा भी
* **स्वचालित फॉलबैक**: अगर Supermemory में समस्या हो, तो अनुरोध सीधे पास-थ्रू हो जाते हैं

<div id="for-applications">
  ### अनुप्रयोगों के लिए
</div>

* **लंबी बातचीत बेहतर**: हजारों संदेशों के बाद भी संदर्भ बनाए रखता है
* **सुसंगत प्रतिक्रियाएँ**: संग्रहीत मेमोरीज़ सत्रों में जानकारी की एकरूपता सुनिश्चित करती हैं
* **स्मार्ट रिट्रीवल**: केवल प्रासंगिक संदर्भ शामिल होता है, जिससे प्रतिक्रिया की गुणवत्ता बेहतर होती है
* **लागत बचत**: स्वचालित चंकिंग से टोकन उपयोग में उल्लेखनीय कमी आती है

<Tabs>

    * **लंबी बातचीत**: वे सेशन जो मॉडल के कॉन्टेक्स्ट विंडो से लंबे हों
    * **मल्टी-सेशन memory**: वे उपयोगकर्ता जो लौटकर बातचीत जारी रखते हैं
    * **त्वरित प्रोटोटाइप**: इंफ्रास्ट्रक्चर बनाए बिना memory क्षमताएँ पाएं

<div id="authentication">
  ## प्रमाणीकरण
</div>

Memory Router को दो API कुंजियों की आवश्यकता होती है:

1. **Supermemory API कुंजी**: memory प्रबंधन के लिए
2. **Provider API कुंजी**: आपके चुने हुए LLM provider के लिए

आप इन्हें निम्न तरीकों से प्रदान कर सकते हैं:

* हेडर्स (प्रोडक्शन के लिए अनुशंसित)
* url पैरामीटर्स (टेस्टिंग के लिए उपयोगी)
* रिक्वेस्ट बॉडी (अनुकूलता के लिए)

<div id="how-memories-work">
  ## मेमोरीज़ कैसे काम करती हैं
</div>

Memory Router का उपयोग करते समय:

1. **स्वचालित निष्कर्षण**: वार्तालापों से महत्वपूर्ण जानकारी अपने‑आप निकाली जाती है
2. **इंटेलिजेंट चंकिंग**: लंबे संदेशों को सैमांटिक चंक में विभाजित किया जाता है
3. **रिलेशनशिप बिल्डिंग**: नई मेमोरीज़ मौजूदा ज्ञान से जुड़ती हैं
4. **स्मार्ट रिट्रीवल**: संदर्भ में केवल सबसे प्रासंगिक मेमोरीज़ शामिल की जाती हैं

<Note>
  जब एक ही `user_id` का उपयोग किया जाता है, तो Memory Router और Memory API के बीच मेमोरीज़ साझा होती हैं, जिससे आप दोनों को साथ में उपयोग कर सकते हैं।
</Note>

<div id="response-headers">
  ## Response Headers
</div>

Memory Router निदान हेडर जोड़ता है ताकि आप समझ सकें कि क्या हो रहा है:

| Header | Description |
|--------|-------------|
| `x-supermemory-conversation-id` | अद्वितीय वार्तालाप पहचानकर्ता |
| `x-supermemory-context-modified` | क्या संदर्भ संशोधित हुआ (`true`/`false`) |
| `x-supermemory-tokens-processed` | प्रोसेस किए गए टोकन की संख्या |
| `x-supermemory-chunks-created` | नए memory चंक बनाए गए |
| `x-supermemory-chunks-retrieved` | संदर्भ में जोड़े गए memory चंक |

<div id="error-handling">
  ## Error Handling
</div>

Memory Router को विश्वसनीयता के लिए डिज़ाइन किया गया है:

* **Automatic Fallback**: यदि Supermemory में कोई त्रुटि आती है, तो आपका अनुरोध बिना किसी बदलाव के पास-थ्रू हो जाता है
* **Error Headers**: `x-supermemory-error` हेडर त्रुटि का विवरण देता है
* **Zero Downtime**: भले ही memory सुविधाएँ उपलब्ध न हों, आपकी एप्लिकेशन सामान्य रूप से चलती रहती है
