तकनीकी आर्किटेक्चर **सादगी और उपयोगकर्ता अनुभव** को प्राथमिकता देता है, साथ ही मज़बूत कार्यक्षमता बनाए रखता है। रचनाकारों के शब्दों में, इसे “सबसे सरल चीज़ जो आप देखेंगे” की तरह बनाया गया है — मूलतः एक React Router एप्लिकेशन जो supermemory API पर fetch कॉल्स करता है — पूरा सिस्टम लगभग 5 घंटे के वास्तविक कार्य समय में विकसित और शिप किया गया।

<div id="architecture-components">
  ### आर्किटेक्चर कॉम्पोनेंट्स
</div>

- **बैकएंड API**: Supermemory API (https://api.supermemory.ai/v3) के ऊपर बनाया गया
- **ट्रांसपोर्ट लेयर**: रियल‑टाइम संचार के लिए Server-Sent Events (SSE) का उपयोग करता है
- **डायनेमिक सर्वर जेनरेशन**: URL पाथ पैरामीटर्स के माध्यम से प्रत्येक उपयोगकर्ता के लिए यूनिक MCP सर्वर इंस्टेंसेज़ बनाता है
- **सेशन मैनेजमेंट**: यूनिक URLs के माध्यम से पूर्ण उपयोगकर्ता आइसोलेशन बनाए रखता है
- **इन्फ्रास्ट्रक्चर**: लंबे समय तक चलने वाले, पर्सिस्टेंट कनेक्शंस के लिए Durable Objects का उपयोग करते हुए Cloudflare पर होस्ट किया गया

यह सिस्टम **Cloudflare के इन्फ्रास्ट्रक्चर** और CPU‑आधारित बिलिंग का लाभ उठाता है, जिससे यह अत्यधिक कुशल हो जाता है क्योंकि memory कनेक्शंस सक्रिय प्रोसेसिंग के बजाय इंटरैक्शंस के बीच अधिकांश समय प्रतीक्षा में बिताते हैं, जिसके परिणामस्वरूप संभावित रूप से लाखों मिलीसेकंड तक चलने पर भी न्यूनतम CPU उपयोग होता है।

<div id="the-two-main-components-explained">
  ## दो मुख्य घटकों का विवरण
</div>

<div id="addtosupermemory-action">
  ### addToSupermemory action
</div>

यह कॉम्पोनेंट उन्नत ट्रिगरिंग मैकेनिज़्म के साथ **उपयोगकर्ता जानकारी, प्राथमिकताएँ और व्यवहार पैटर्न** को संग्रहित करता है:

**ट्रिगर विधियाँ:**

- **स्पष्ट कमांड**: "remember this" जैसे सीधे उपयोगकर्ता निर्देश
- **अप्रत्यक्ष पहचान**: बातचीत के दौरान महत्वपूर्ण उपयोगकर्ता गुण, प्राथमिकताएँ या पैटर्न की स्वचालित पहचान

**कैप्चर किए गए डेटा प्रकार:**

- तकनीकी प्राथमिकताएँ और विवरण (उदा., "My primary programming language is Python")
- प्रोजेक्ट संबंधी जानकारी और संदर्भ (उदा., "I'm currently working on a project named 'Apollo'")
- उपयोगकर्ता व्यवहार और भावनात्मक प्रतिक्रियाएँ
- व्यक्तिगत तथ्य, प्राथमिकताएँ और निर्णय लेने के पैटर्न
- तकनीकी विवरण और उदाहरण सहित समृद्ध संदर्भ

<div id="searchsupermemory-action">
  ### searchSupermemory action
</div>

यह घटक उन्नत खोज क्षमताओं का उपयोग करके संग्रहीत स्मृतियों से **प्रासंगिक जानकारी प्राप्त करता है**:

**सक्रियण ट्रिगर्स:**

- ऐतिहासिक जानकारी के लिए स्पष्ट उपयोगकर्ता अनुरोध
- ऐसी संदर्भगत स्थितियाँ जहाँ पिछले उपयोगकर्ता चुनाव वर्तमान निर्णयों में सहायक हों
- वार्तालाप के प्रवाह के आधार पर स्वचालित संदर्भ संवर्द्धन

**खोज क्षमताएँ:**

- **सामन्तिक मिलान**: वेक्टर खोज का उपयोग करके संबंधित अनुभवों में प्रासंगिक विवरण ढूँढता है
- **पैटर्न पहचान**: व्यवहार पैटर्न और प्राथमिकताओं की पहचान करता है
- **क्रॉस-सेशन रिट्रीवल**: पिछली वार्तालापों और प्लेटफ़ॉर्मों से memory तक पहुँचता है
- **बुद्धिमान फ़िल्टरिंग**: वर्तमान वार्तालाप की आवश्यकताओं के आधार पर सबसे प्रासंगिक संदर्भ लौटाता है
