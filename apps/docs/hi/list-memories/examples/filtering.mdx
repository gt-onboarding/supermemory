container tags और मेटाडेटा के आधार पर सहेजी गई स्मृतियों को फ़िल्टर करें। फ़िल्टरिंग सिस्टम SQL query construction का उपयोग करता है, इसलिए आपको अपने filters को database queries की तरह संरचित करना होगा।

<div id="filter-by-container-tags">
  ## Container Tags के आधार पर फ़िल्टर करें
</div>

Container tags सटीक array matching का उपयोग करते हैं — memories में बिल्कुल वही टैग उसी क्रम में होने चाहिए।

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    // Single tag - बिल्कुल ["user_123"] वाली memories से मेल खाता है
    const userMemories = await client.memories.list({
      containerTags: ["user_123"]
    });

    // Multiple tags - बिल्कुल ["user_123", "project_ai"] वाली memories से मेल खाता है
    const projectMemories = await client.memories.list({
      containerTags: ["user_123", "project_ai"]
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    # Single tag
    user_memories = client.memories.list(container_tags=["user_123"])

    # Multiple tags (exact match)
    project_memories = client.memories.list(
        container_tags=["user_123", "project_ai"]
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    # Single tag
    curl -X POST "https://api.supermemory.ai/v3/documents/list" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"containerTags": ["user_123"]}'

    # Multiple tags
    curl -X POST "https://api.supermemory.ai/v3/documents/list" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
    ```

<div id="metadata-filtering-with-sql-logic">
  ## SQL लॉजिक के साथ मेटाडेटा फ़िल्टरिंग
</div>

`filters` पैरामीटर SQL-जैसी क्वेरी संरचनाओं का उपयोग करके मेटाडेटा फ़ील्ड्स पर फ़िल्टर लगाने की सुविधा देता है। चूंकि हम बैकएंड में SQL क्वेरी निर्माण का उपयोग करते हैं, इसलिए आपको अपने filters को स्पष्ट AND/OR लॉजिक के साथ डेटाबेस क्वेरियों की तरह संरचित करना होगा।

<div id="why-this-structure">
  ### यह संरचना क्यों?
</div>

SQL डेटाबेस में `AND` की प्राथमिकता `OR` से अधिक होती है। स्पष्ट समूहबद्धता के बिना, इस तरह की क्वेरी:

```
category = 'programming' OR framework = 'react' AND difficulty = 'advanced'
```

का अर्थ इस प्रकार समझा जाता है:

```
category = 'programming' OR (framework = 'react' AND difficulty = 'advanced')
```

The JSON संरचना अनपेक्षित परिणामों से बचने के लिए स्पष्ट समूहबद्धता लागू करती है।

<Info>
  **फ़िल्टर संरचना नियम:**

  * हमेशा शर्तों को `AND` या `OR` एरे में रखें (भले ही केवल एक ही शर्त हो)
  * फ़िल्टर ऑब्जेक्ट को स्ट्रिंग में बदलने के लिए `JSON.stringify()` का उपयोग करें
  * प्रत्येक शर्त में `key`, `value`, और `negate` प्रॉपर्टीज़ होनी चाहिए
  * सामान्य मिलान के लिए `negate: false`, और बाहर करने के लिए `negate: true`
</Info>


<div id="simple-metadata-filter">
  ### सरल मेटाडेटा फ़िल्टर
</div>

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    // एकल मेटाडेटा फ़ील्ड से फ़िल्टर करें
    const programmingMemories = await client.memories.list({
      filters: JSON.stringify({
        AND: [
          { key: "category", value: "programming", negate: false }
        ]
      })
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import json

    # एकल मेटाडेटा फ़ील्ड से फ़िल्टर करें
    programming_memories = client.memories.list(
        filters=json.dumps({
            "AND": [
                {"key": "category", "value": "programming", "negate": False}
            ]
        })
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/documents/list" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "filters": "{\"AND\":[{\"key\":\"category\",\"value\":\"programming\",\"negate\":false}]}"
      }'
    ```
  </Tab>
</Tabs>

<Warning>
**आम गलतियाँ:**
- अकेले कंडीशन ऑब्जेक्ट का उपयोग करना: `{"key": "category", "value": "programming"}`
- JSON.stringify भूल जाना: स्ट्रिंग की जगह ऑब्जेक्ट पास करना
- negate प्रॉपर्टी छोड़ देना: हमेशा `"negate": false` या `"negate": true` शामिल करें
</Warning>

<Note>
**Container Tags बनाम Metadata फ़िल्टरिंग:**
- Container tags: संगठनात्मक समूह के लिए सटीक ऐरे मिलान
- Metadata filters: कस्टम मेटाडेटा फ़ील्ड्स पर जटिल लॉजिक के साथ SQL-जैसी क्वेरी
- दोनों को मिलाकर शक्तिशाली फ़िल्टरिंग क्षमताएँ मिलती हैं
</Note>
