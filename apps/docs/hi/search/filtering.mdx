Supermemory `/v3/search` और `/v4/search` दोनों endpoints के लिए कंटेनर टैग, मेटाडेटा शर्तें, और उन्नत फिल्टरिंग तकनीकों के माध्यम से खोज परिणामों को फिल्टर करने का समर्थन करता है।

<div id="container-tags">
  ## कंटेनर टैग्स
</div>

कंटेनर टैग उपयोगकर्ता, प्रोजेक्ट या संगठन के अनुसार संग्रहीत मेमोरियाँ को समूहित करते हैं। खोज परिणामों को अलग रखने का यह प्राथमिक तरीका है।

**महत्वपूर्ण**: कंटेनर टैग्स **exact array matching** का उपयोग करते हैं। `["technology", "quantum-computing"]` वाला कोई दस्तावेज़ `["technology"]` की खोज से मेल नहीं खाएगा। arrays बिलकुल समान होने चाहिए।

<div id="document-search-v3search">
  ### दस्तावेज़ खोज (v3/search)
</div>

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    // एक container tag
    const results = await client.search.documents({
      q: "machine learning",
      containerTags: ["user_123"],
      limit: 10
    });

    // कई container tags
    const results2 = await client.search.documents({
      q: "project status",
      containerTags: ["project_ai", "team_research"],
      limit: 10
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    # एक container tag
    results = client.search.documents(
        q="machine learning",
        container_tags=["user_123"],
        limit=10
    )

    # कई container tags
    results2 = client.search.documents(
        q="project status",
        container_tags=["project_ai", "team_research"],
        limit=10
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    # एक container tag
    curl -X POST "https://api.supermemory.ai/v3/search" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "q": "machine learning",
        "containerTags": ["user_123"],
        "limit": 10
      }'
    ```
  </Tab>
</Tabs>

    ```bash
        "limit": 5
      }'
    ```
  </Tab>

<Note>
**Container Tag में अंतर**:
- `/v3/search` `containerTags` (बहुवचन array) का उपयोग करता है, जिसमें exact array matching होता है
- `/v4/search` एकल टैग फ़िल्टरिंग के लिए `containerTag` (एकवचन string) का उपयोग करता है
- Exact matching का मतलब है `["user", "project"]` ≠ `["user"]`
</Note>

<div id="filter-best-practices">
  ## फ़िल्टर के सर्वोत्तम तरीके
</div>

<Callout type="info">
**प्रदर्शन संबंधित टिप्स**:
- प्राथमिक फ़िल्टर के रूप में कंटेनर टैग्स का उपयोग करें (सबसे तेज़)
- अधिक सटीकता के लिए सरल मेटाडेटा filters के साथ संयोजन करें
- बेहतर प्रदर्शन के लिए जटिल नेस्टेड OR शर्तों से बचें
- केवल आवश्यक होने पर ही संख्यात्मक ऑपरेटरों का उपयोग करें
</Callout>

<Warning>
**फ़िल्टर संरचना**: सभी शर्तों को `AND` या `OR` arrays में लपेटा जाना चाहिए। एक एकल शर्त को भी array रैपर की आवश्यकता होती है: `{"AND": [{"key": "category", "value": "ai"}]}`.
</Warning>

<Note>
**समर्थित ऑपरेटर**:
- **स्ट्रिंग**: केवल सटीक मिलान
- **संख्यात्मक**: `<=`, `>=`, `<`, `>`, `=`
- **एरे**: `array_contains` — यह जांचने के लिए कि एरे में मान शामिल है या नहीं
- **निषेध (Negation)**: सभी filter प्रकारों के साथ काम करता है
</Note>
