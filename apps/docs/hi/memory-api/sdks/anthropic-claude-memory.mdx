Supermemory को स्थायी संग्रहण बैकएंड के रूप में उपयोग करके Claude की मूल memory टूल कार्यक्षमता सक्षम करें। Claude परिचित फ़ाइल सिस्टम ऑपरेशंस का उपयोग करते हुए बातचीतों के बीच स्वचालित रूप से जानकारी संग्रहीत कर सकता है और पुनर्प्राप्त कर सकता है।

<div id="quick-start">
  ## त्वरित शुरुआत
</div>

```typescript
import Anthropic from "@anthropic-ai/sdk"
import { createClaudeMemoryTool } from "@supermemory/tools/claude-memory"

const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY!,
})

const memoryTool = createClaudeMemoryTool(process.env.SUPERMEMORY_API_KEY!, {
  projectId: 'my-app',
})

async function chatWithMemory(userMessage: string) {
  const response = await anthropic.beta.messages.create({
    model: 'claude-sonnet-4-5',
    max_tokens: 8096,
    tools: [{ type: 'memory_20250818', name: 'memory' }],
    betas: ['context-management-2025-06-27'],
    messages: [{ role: 'user', content: userMessage }],
  })

  // Memory tool calls को handle करें
  for (const block of response.content) {
    if (block.type === 'tool_use' && block.name === 'memory') {
      const result = await memoryTool.handleCommandForToolResult(
        block.input as MemoryCommand,
        block.id
      )
      console.log('Memory operation result:', result)
    }
  }

  return response
}
```

<div id="how-it-works">
  ## यह कैसे काम करता है
</div>

Claude का memory टूल स्थायी जानकारी प्रबंधन के लिए एक फाइलसिस्टम रूपक का उपयोग करता है:

* **Files**: व्यक्तिगत memory आइटम जो Supermemory Documents के रूप में संग्रहीत होते हैं
* **Directories**: पाथ संरचना का उपयोग करके व्यवस्थित (उदा., `/memories/user-preferences`)
* **Operations**: फाइलों को देखना, बनाना, संपादित करना, हटाना और नाम बदलना
* **Path Normalization**: `/memories/preferences` जैसे पाथ `--memories--preferences` के रूप में संग्रहीत किए जाते हैं

<div id="complete-chat-example">
  ## पूर्ण चैट उदाहरण
</div>

यहाँ बातचीत हैंडलिंग के साथ एक संपूर्ण उदाहरण है:

```typescript
import Anthropic from "@anthropic-ai/sdk"
import { createClaudeMemoryTool, type MemoryCommand } from "@supermemory/tools/claude-memory"

const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY!,
})

const memoryTool = createClaudeMemoryTool(process.env.SUPERMEMORY_API_KEY!, {
  projectId: 'chat-app',
})

async function chat() {
  const messages: Anthropic.MessageParam[] = []

  // पहला संदेश: जानकारी स्टोर करें
  messages.push({
    role: 'user',
    content: 'याद रखें कि मैं डार्क मोड पसंद करता हूं और TypeScript का उपयोग करता हूं'
  })

  let response = await anthropic.beta.messages.create({
    model: 'claude-sonnet-4-5',
    max_tokens: 8096,
    tools: [{ type: 'memory_20250818', name: 'memory' }],
    betas: ['context-management-2025-06-27'],
    messages,
  })

  // टूल कॉल्स को प्रोसेस करें
  const toolResults: Anthropic.ToolResultBlockParam[] = []

  for (const block of response.content) {
    if (block.type === 'tool_use' && block.name === 'memory') {
      const result = await memoryTool.handleCommandForToolResult(
        block.input as MemoryCommand,
        block.id
      )
      toolResults.push(result)
    }
  }

  // टूल परिणामों के साथ बातचीत जारी रखें
  if (toolResults.length > 0) {
    messages.push({ role: 'assistant', content: response.content })
    messages.push({ role: 'user', content: toolResults })

    response = await anthropic.beta.messages.create({
      model: 'claude-sonnet-4-5',
      max_tokens: 8096,
      tools: [{ type: 'memory_20250818', name: 'memory' }],
      betas: ['context-management-2025-06-27'],
      messages,
    })
  }

  console.log(response.content)
}

chat()
```

  ## API संदर्भ
</div>

<div id="createclaudememorytoolapikey-config">
  ### `createClaudeMemoryTool(apiKey, config)`
</div>

Claude के लिए एक memory टूल इंस्टेंस बनाता है।

**पैरामीटर्स:**

* `apiKey` (string): आपकी Supermemory API कुंजी
* `config` (optional):
  * `projectId` (string): प्रोजेक्ट पहचानकर्ता
  * `containerTags` (string[]): `projectId` का विकल्प
  * `baseUrl` (string): कस्टम API endpoint

**रिटर्न्स:** `ClaudeMemoryTool` इंस्टेंस

<div id="handlecommandfortoolresultcommand-tooluseid">
  ### `handleCommandForToolResult(command, toolUseId)`
</div>

एक memory कमांड को प्रोसेस करता है और फ़ॉर्मैट किया हुआ परिणाम लौटाता है।

**Parameters:**

* `command` (MemoryCommand): Claude से आने वाला memory ऑपरेशन
* `toolUseId` (string): Claude के response से tool use ID

**Returns:** `Promise<MemoryToolResult>` जिसमें:

* `type`: &quot;tool&#95;result&quot;
* `tool_use_id`: tool use ID
* `content`: ऑपरेशन का परिणाम या त्रुटि संदेश
* `is_error`: सफलता/विफलता दर्शाने वाला boolean

<div id="memory-commands">
  ## मेमोरी कमांड्स
</div>

Claude आंतरिक रूप से इन कमांड प्रकारों का उपयोग करता है:

| कमांड | विवरण | उदाहरण |
|---------|-------------|---------|
| `view` | डायरेक्टरी सूचीबद्ध करें या फ़ाइल पढ़ें | `/memories/` या `/memories/file.txt` |
| `create` | नई फ़ाइल बनाएं | सामग्री सहित `/memories/new.txt` बनाएं |
| `str_replace` | फ़ाइल की सामग्री संपादित करें | &quot;old text&quot; को &quot;new text&quot; से बदलें |
| `insert` | निर्दिष्ट पंक्ति पर सामग्री जोड़ें | पंक्ति 5 पर जोड़ें |
| `delete` | फ़ाइल हटाएं | `/memories/old.txt` हटाएं |
| `rename` | फ़ाइल का नाम बदलें/स्थानांतरित करें | `old.txt` का नाम बदलकर `new.txt` करें |

<div id="when-to-use">
  ## कब उपयोग करें
</div>

Claude Memory Tool इन उपयोग मामलों के लिए आदर्श है:

* **Conversational AI**: Claude स्वतः उपयोगकर्ता की पसंद और संदर्भ याद रखता है
* **Personal Assistants**: उपयोगकर्ता-विशिष्ट जानकारी सहेजें और पुनर्प्राप्त करें
* **Documentation Bots**: वार्तालापों के बीच ज्ञान बनाए रखें
* **Project Management**: कार्यों और प्रोजेक्ट की स्थिति पर नज़र रखें
* **Note-Taking Apps**: मीटिंग नोट्स और विचारों के लिए स्थायी memory

<div id="limitations">
  ## सीमाएँ
</div>

* बीटा फीचर्स सक्षम होने के साथ Anthropic SDK आवश्यक
* पथ सेपरेटर (`/`) को स्टोरेज के लिए `--` में सामान्यीकृत किया जाता है
* प्रति डायरेक्टरी लिस्टिंग अधिकतम 100 फ़ाइलें
* फ़ाइल ऑपरेशन असिंक्रोनस होते हैं
