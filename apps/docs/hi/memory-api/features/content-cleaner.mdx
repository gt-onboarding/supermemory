supermemory आपकी कंटेंट प्रोसेसिंग पाइपलाइन को अनुकूलित करने के लिए उन्नत कॉन्फ़िगरेशन विकल्प प्रदान करता है। इसके केंद्र में एक AI-संचालित सिस्टम है जो आपकी विशिष्ट आवश्यकताओं के अनुसार आपकी सामग्री का स्वचालित रूप से विश्लेषण, श्रेणीकरण और फ़िल्टर कर सकता है।

<div id="configuration-schema">
  ## कॉन्फ़िगरेशन स्कीमा
</div>

```json
{
  "shouldLLMFilter": true,
  "categories": ["feature-request", "bug-report", "positive", "negative"],
  "filterPrompt": "Analyze feedback sentiment and identify feature requests",
  "includeItems": ["critical", "high-priority"],
  "excludeItems": ["spam", "irrelevant"]
}
```


<div id="core-settings">
  ## मुख्य सेटिंग्स
</div>

<div id="shouldllmfilter">
  ### shouldLLMFilter
</div>

- **प्रकार**: `boolean`
- **आवश्यक**: नहीं (डिफ़ॉल्ट: `false`)
- **विवरण**: AI-संचालित सामग्री विश्लेषण के लिए मुख्य स्विच। किसी भी उन्नत फ़िल्टरिंग सुविधा का उपयोग करने हेतु इसे सक्षम करना ज़रूरी है।

<div id="categories">
  ### categories
</div>

- **प्रकार**: `string[]`
- **सीमाएँ**: प्रत्येक श्रेणी 1–50 वर्णों की होनी चाहिए
- **अनिवार्य**: नहीं
- **विवरण**: सामग्री वर्गीकरण के लिए कस्टम श्रेणियाँ निर्धारित करें। यदि निर्दिष्ट की गई हैं, तो AI केवल इन्हीं श्रेणियों का उपयोग करेगा। यदि निर्दिष्ट नहीं हैं, तो यह स्वचालित रूप से 3–5 प्रासंगिक श्रेणियाँ उत्पन्न करेगा।

<div id="filterprompt">
  ### filterPrompt
</div>

- **प्रकार**: `string`
- **सीमा**: 1–750 वर्ण
- **अनिवार्य**: नहीं
- **विवरण**: सामग्री का विश्लेषण और श्रेणीकरण करने के तरीके पर AI के लिए कस्टम निर्देश। अपनी विशिष्ट आवश्यकताओं के अनुसार श्रेणीकरण प्रक्रिया को निर्देशित करने के लिए इसका उपयोग करें।

<div id="includeitems-excludeitems">
  ### includeItems & excludeItems
</div>

- **प्रकार**: `string[]`
- **सीमाएँ**: प्रत्येक आइटम 1–20 अक्षरों का होना चाहिए
- **आवश्यक**: नहीं
- **विवरण**: प्रोसेसिंग के दौरान जिन आइटमों को स्पष्ट रूप से शामिल या बाहर करना है, उन्हें निर्दिष्ट करके सामग्री फ़िल्टरिंग को सूक्ष्म रूप से नियंत्रित करें।

<div id="content-processing-pipeline">
  ## कंटेंट प्रोसेसिंग पाइपलाइन
</div>

जब LLM फ़िल्टरिंग सक्षम हो, तो कंटेंट इनजेस्ट होने पर:

1. **प्रारंभिक प्रोसेसिंग**
   - कंटेंट को निकाला और सामान्यीकृत किया जाता है
   - बुनियादी metadata (title, description) कैप्चर किया जाता है

2. **AI विश्लेषण**
   - कंटेंट का आपके `filterPrompt` के आधार पर विश्लेषण किया जाता है
   - श्रेणियाँ असाइन की जाती हैं (आपकी पूर्वनिर्धारित सूची से या स्वत: जनरेटेड)
   - टैग्स का मूल्यांकन कर उन्हें अंक दिए जाते हैं

3. **चंकिंग और इंडेक्सिंग**
   - कंटेंट को सेमान्टिक चंक में विभाजित किया जाता है
   - प्रत्येक चंक को कुशल खोज के लिए एम्बेड किया जाता है
   - metadata और वर्गीकरण संग्रहीत किए जाते हैं

<div id="example-use-cases">
  ## उदाहरण उपयोग-मामले
</div>

<div id="1-customer-feedback-system">
  ### 1. ग्राहक प्रतिक्रिया प्रणाली
</div>

```json
{
  "shouldLLMFilter": true,
  "categories": ["positive", "negative", "neutral"],
  "filterPrompt": "ग्राहक भावना का विश्लेषण करें और मुख्य विषयों की पहचान करें",
}
```


<div id="2-content-moderation">
  ### 2. सामग्री मॉडरेशन
</div>

```json
{
  "shouldLLMFilter": true,
  "categories": ["सुरक्षित", "समीक्षा-आवश्यक", "चिह्नित"],
  "filterPrompt": "संभावित अनुचित या संवेदनशील सामग्री की पहचान करें",
  "excludeItems": ["स्पैम", "आपत्तिजनक"],
  "includeItems": ["उपयोगकर्ता-जनित"]
}
```

> **महत्वपूर्ण**: सभी फ़िल्टरिंग फीचर्स (`categories`, `filterPrompt`, `includeItems`, `excludeItems`) के लिए `shouldLLMFilter` को सक्षम करना आवश्यक है। `shouldLLMFilter` को सक्षम किए बिना इन फीचर्स का उपयोग करने पर 400 त्रुटि मिलेगी।
