Monitore o uso do Supermemory com análises detalhadas de chamadas à API, erros e métricas de desempenho.

<div id="overview">
  ## Visão geral
</div>

A API de Analytics fornece insights abrangentes sobre o uso da sua Supermemory:

- **Estatísticas de uso**: Acompanhe chamadas de API por tipo, tendências por hora e detalhamento por chave de API
- **Monitoramento de erros**: Identifique os principais tipos de erro e padrões
- **Logs detalhados**: Acesse logs completos de requisição/resposta para depuração
- **Métricas de desempenho**: Monitore tempos médios de resposta e duração do processamento

<Note>
Os dados de analytics estão disponíveis para toda a sua organização e podem ser filtrados por período.
</Note>

<div id="usage-statistics">
  ## Estatísticas de uso
</div>

Obtenha estatísticas de uso abrangentes, incluindo detalhamento por hora e métricas por chave.

<div id="endpoint">
  ### Endpoint
</div>

`GET /v3/analytics/usage`

<div id="parameters">
  ### Parâmetros
</div>

| Parâmetro | Tipo | Descrição |
|-----------|------|-------------|
| `from` | string (ISO 8601) | Data/hora de início do período |
| `to` | string (ISO 8601) | Data/hora de término do período |
| `period` | string | Alternativa a `from`: `1h`, `24h`, `7d`, `30d` |
| `page` | integer | Número da página para paginação (padrão: 1) |
| `limit` | integer | Itens por página (padrão: 20, máx.: 100) |

<div id="example-request">
  ### Exemplo de requisição
</div>

<CodeGroup>

```typescript TypeScript
// Obter o uso das últimas 24 horas
const usage = await fetch('https://api.supermemory.ai/v3/analytics/usage?period=24h', {
  headers: {
    'Authorization': `Bearer ${SUPERMEMORY_API_KEY}`
  }
});

const data = await usage.json();
```

```python Python
import requests
from datetime import datetime, timedelta

# Obter o uso dos últimos 7 dias
response = requests.get(
    'https://api.supermemory.ai/v3/analytics/usage',
    params={'period': '7d'},
    headers={'Authorization': f'Bearer {SUPERMEMORY_API_KEY}'}
)

data = response.json()
```

```bash cURL
# Obter o uso para um intervalo de datas específico
curl -X GET "https://api.supermemory.ai/v3/analytics/usage?from=2024-01-01T00:00:00Z&to=2024-01-31T23:59:59Z" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"
```

</CodeGroup>

<div id="error-monitoring">
  ## Monitoramento de erros
</div>

Acompanhe e analise os erros para identificar problemas e melhorar a confiabilidade.

<div id="endpoint">
  ### Endpoint
</div>

`GET /v3/analytics/errors`

<div id="parameters">
  ### Parâmetros
</div>

Os mesmos do endpoint de uso — oferece suporte a `from`, `to`, `period`, `page` e `limit`.

<div id="example-request">
  ### Exemplo de requisição
</div>

<CodeGroup>

```typescript TypeScript
// Obter erros das últimas 24 horas
const errors = await fetch('https://api.supermemory.ai/v3/analytics/errors?period=24h', {
  headers: {
    'Authorization': `Bearer ${SUPERMEMORY_API_KEY}`
  }
});

const data = await errors.json();
```

```python Python
# Monitorar erros e emitir alertas em caso de picos
response = requests.get(
    'https://api.supermemory.ai/v3/analytics/errors?period=1h',
    headers={'Authorization': f'Bearer {SUPERMEMORY_API_KEY}'}
)

data = response.json()
```

```bash cURL
# Obter erros dos últimos 7 dias
curl -X GET "https://api.supermemory.ai/v3/analytics/errors?period=7d" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"
```

</CodeGroup>

<div id="detailed-logs">
  ## Logs detalhados
</div>

Acesse logs completos de requisição/resposta para depuração e auditoria.

<div id="endpoint">
  ### Endpoint
</div>

`GET /v3/analytics/logs`

<div id="parameters">
  ### Parâmetros
</div>

Os mesmos do endpoint de uso, com filters opcionais:

- `type`: Filtrar pelo tipo de requisição (add, search, update, delete)
- `statusCode`: Filtrar pelo código de status HTTP
- `keyId`: Filtrar por id de uma chave de API específica

<div id="example-request">
  ### Exemplo de requisição
</div>

<CodeGroup>

```typescript TypeScript
// Obter solicitações recentes com erro
const logs = await fetch('https://api.supermemory.ai/v3/analytics/logs?period=1h&statusCode=500', {
  headers: {
    'Authorization': `Bearer ${SUPERMEMORY_API_KEY}`
  }
});

const data = await logs.json();
```

```python Python
# Depurar uso específico da chave de API
response = requests.get(
    'https://api.supermemory.ai/v3/analytics/logs',
    params={
        'keyId': 'key_abc123',
        'period': '24h'
    },
    headers={'Authorization': f'Bearer {SUPERMEMORY_API_KEY}'}
)

logs = response.json()['logs']

```

```bash cURL
# Obter todos os logs para depuração
curl -X GET "https://api.supermemory.ai/v3/analytics/logs?period=1h&limit=50" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"
```

</CodeGroup>

<div id="rate-limits">
  ## Limites de taxa
</div>

Os endpoints de Analytics têm os seguintes limites:

- 100 solicitações por minuto por organização
- Intervalo máximo: 90 dias
- Tamanho máximo da página: 100 itens

<Warning>
Os dados de Analytics são mantidos por 90 dias. Para retenções mais longas, exporte e armazene os dados nos seus próprios sistemas.
</Warning>
