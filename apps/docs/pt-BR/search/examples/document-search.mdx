A busca em documents (`POST /v3/search`) oferece controle máximo sobre o comportamento da pesquisa, com parâmetros abrangentes para ajuste fino dos resultados.

<div id="basic-implementation">
  ## Implementação básica
</div>

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    import Supermemory from 'supermemory';

    const client = new Supermemory({
      apiKey: process.env.SUPERMEMORY_API_KEY!
    });

    const results = await client.search.documents({
      q: "machine learning neural networks",
      limit: 5
    });

    console.log(`Foram encontrados ${results.total} documents em ${results.timing}ms`);

    // Exemplo de estrutura de saída
    results.results.forEach((doc, i) => {
      console.log(`${i + 1}. ${doc.title} (score: ${doc.score})`);
      console.log(`   ${doc.chunks.length} chunks encontrados`);
    });
    ```
  </Tab>

  <Tab title="Python">
    ```python
    from supermemory import Supermemory
    import os

    client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

    results = client.search.documents(
        q="machine learning neural networks",
        limit=5
    )

    print(f"Foram encontrados {results.total} documents em {results.timing}ms")

    # Exemplo de estrutura de saída
    for i, doc in enumerate(results.results):
        print(f"{i + 1}. {doc.title} (score: {doc.score})")
        print(f"   {len(doc.chunks)} chunks encontrados")
    ```
  </Tab>

  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/search" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "q": "machine learning neural networks",
        "limit": 5
      }'
    ```
  </Tab>
</Tabs>

<div id="container-tags-filtering">
  ## Filtragem por tags de container
</div>

As tags de container são a principal forma de isolar resultados de busca por usuário, projeto ou organização.

**Comportamentos principais:**

- **Baseado em array**: Diferentemente de `/v4/search`, este endpoint aceita várias tags de container em um array
- **Correspondência exata do array**: Os documents devem ter exatamente o mesmo array de tags de container para corresponder

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const results = await client.search.documents({
      q: "quarterly reports",
      containerTags: ["user_123"],
      limit: 10
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    results = client.search.documents(
        q="quarterly reports",
        container_tags=["user_123"],
        limit=10
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/search" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "q": "quarterly reports",
        "containerTags": ["user_123"],
        "limit": 10
      }'
    ```
  </Tab>
</Tabs>

<div id="metadata-filtering">
  ## Filtragem por metadata
</div>

A filtragem por metadata permite criar condições complexas sobre dados estruturados anexados aos seus documents. Isso utiliza construção de consultas ao estilo SQL no backend, exigindo estruturas explícitas com AND/OR.

**Regras da estrutura de filtros:**

* **As condições devem estar envolvidas** em arrays AND ou OR, mesmo quando houver apenas uma condição
* **Suporta correspondência de strings** (exata), operadores numéricos e verificação de conteúdo em arrays
* **Inverta qualquer condição** com `negate: true`
* **Combina com tags de contêiner** — ambos os filters são aplicados

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const results = await client.search.documents({
      q: "machine learning",
      filters: {
        AND: [
          {
            key: "category",
            value: "technology",
            negate: false
          },
          {
            filterType: "numeric",
            key: "readingTime",
            value: "5",
            negate: false,
            numericOperator: "<="
          }
        ]
      },
      limit: 10
    });
    ```
  </Tab>

  <Tab title="Python">
    ```python
    results = client.search.documents(
        q="machine learning",
        filters={
            "AND": [
                {
                    "key": "category",
                    "value": "technology",
                    "negate": False
                },
                {
                    "filterType": "numeric",
                    "key": "readingTime",
                    "value": "5",
                    "negate": False,
                    "numericOperator": "<="
                }
            ]
        },
        limit=10
    )
    ```
  </Tab>

  <Tab title="cURL">

<div id="array-contains-filtering">
  ## Filtragem com array_contains
</div>

Quando a sua metadata inclui arrays (como listas de participantes, tags ou categorias), use `array_contains` para verificar se o array contém um valor específico.

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const results = await client.search.documents({
      q: "meeting discussion",
      filters: {
        AND: [
          {
            key: "participants",
            value: "john.doe",
            filterType: "array_contains"
          }
        ]
      },
      limit: 5
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    results = client.search.documents(
        q="meeting discussion",
        filters={
            "AND": [
                {
                    "key": "participants",
                    "value": "john.doe",
                    "filterType": "array_contains"
                }
            ]
        },
        limit=5
    )
    ```
  </Tab>
  <Tab title="cURL">

<div id="threshold-control">
  ## Controle de thresholds
</div>

Controle a qualidade dos resultados com thresholds de sensibilidade:

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const results = await client.search.documents({
      q: "artificial intelligence",
      documentThreshold: 0.7,  // Maior = menos documentos, mais relevantes

      limit: 10
    });
    ```

<div id="query-rewriting">
  ## Reescrita de Consulta
</div>

Melhore a precisão da busca com reescrita automática de consultas:

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const results = await client.search.documents({
      q: "What is the capital of France?",
      rewriteQuery: true,  // +400ms de latência, porém resultados melhores
      limit: 5
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    results = client.search.documents(
        q="What is the capital of France?",
        rewrite_query=True,  # +400ms de latência, porém resultados melhores
        limit=5
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/search" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "q": "What is the capital of France?",
        "rewriteQuery": true,
        "limit": 5
      }'
    ```
  </Tab>
</Tabs>

<Note>
A Reescrita de Consulta gera várias variações da consulta e pesquisa por todas elas, depois mescla os resultados. Sem custo adicional, mas adiciona ~400 ms de latência.
</Note>

<div id="full-context-options">
  ## Opções de Contexto Completo
</div>

Incluir o conteúdo completo dos documentos e seus resumos:

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const results = await client.search.documents({
      q: "research findings",
      includeFullDocs: true,    // Incluir conteúdo completo do documento
      includeSummary: true,     // Incluir resumos do documento
      onlyMatchingChunks: false, // Incluir todos os chunks, não apenas os correspondentes
      limit: 5
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    results = client.search.documents(
        q="research findings",
        include_full_docs=True,    # Incluir conteúdo completo do documento
        include_summary=True,      # Incluir resumos do documento
        only_matching_chunks=False, # Incluir todos os chunks, não apenas os correspondentes
        limit=5
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/search" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "q": "research findings",
        "includeFullDocs": true,
        "includeSummary": true,
        "onlyMatchingChunks": false,
        "limit": 5
      }'
    ```
