Escolha entre atualizações diretas, upserts idempotentes, exclusões individuais e operações em lote avançadas.

<div id="direct-updates">
  ## Atualizações diretas
</div>

Atualize memórias existentes pelo ID quando souber exatamente qual memory deseja modificar. As alterações acionam o reprocessamento por todo o pipeline.

<CodeGroup>

```typescript Typescript
import Supermemory from 'supermemory';

const client = new Supermemory({
  apiKey: process.env.SUPERMEMORY_API_KEY!
});

// Update by memory ID
const updated = await client.memories.update('memory_id_123', {
  content: 'Updated content here',
  metadata: { version: 2, updated: true }
});

console.log(updated.status); // "queued" para reprocessamento
console.log(updated.id); // "memory_id_123"
```

```python Python
from supermemory import Supermemory
import os

client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

# Update by memory ID
updated = client.memories.update(
    'memory_id_123',
    content='Updated content here',
    metadata={'version': 2, 'updated': True}
)

print(f"Status: {updated.status}")  # "queued" para reprocessamento
print(f"ID: {updated.id}")  # "memory_id_123"
```

```bash cURL
curl -X PATCH "https://api.supermemory.ai/v3/documents/memory_id_123" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Updated content here",
    "metadata": {"version": 2, "updated": true}
  }'
```

</CodeGroup>

<div id="upserts-using-customid">
  ## Upserts usando customId
</div>

Use `customId` para operações idempotentes em que o mesmo `customId` com `add()` atualizará a memory existente em vez de criar duplicatas.

<CodeGroup>

```typescript Typescript
import Supermemory from 'supermemory';

const client = new Supermemory({
  apiKey: process.env.SUPERMEMORY_API_KEY!
});

const customId = 'user-note-001';

// Primeira chamada cria a memory
const created = await client.memories.add({
  content: 'Initial content',
  customId: customId,
  metadata: { version: 1 }
});

console.log('Memory criada:', created.id);

// Segunda chamada com o mesmo customId atualiza a existente
const updated = await client.memories.add({
  content: 'Updated content',
  customId: customId,         // Mesmo customId = upsert
  metadata: { version: 2 }
});
```

```python Python
from supermemory import Supermemory
import os

client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

custom_id = 'user-note-001'

# Primeira chamada cria a memory
created = client.memories.add(
    content='Initial content',
    custom_id=custom_id,
    metadata={'version': 1}
)

print(f'Memory criada: {created.id}')

# Segunda chamada com o mesmo customId atualiza a existente
updated = client.memories.add(
    content='Updated content',
    custom_id=custom_id,     # Mesmo customId = upsert
    metadata={'version': 2}
)

print(f'Memory atualizada: {updated.id}')
print(f'É a mesma memory? {created.id == updated.id}')  # True
```

```bash cURL
# Primeira chamada — cria a memory
curl -X POST "https://api.supermemory.ai/v3/documents" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Initial content",
    "customId": "user-note-001",
    "metadata": {"version": 1}
  }'

# Resposta: {"id": "mem_abc123", "status": "queued", "customId": "user-note-001"}

# Segunda chamada — atualiza a existente (mesmo customId)
curl -X POST "https://api.supermemory.ai/v3/documents" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Updated content",
    "customId": "user-note-001",
    "metadata": {"version": 2}
  }'

# Resposta: {"id": "mem_abc123", "status": "queued", "customId": "user-note-001"}
# Observação: o mesmo ID foi retornado — a memory foi atualizada, não criada
```

</CodeGroup>

<Note>
O `customId` habilita idempotência em todos os endpoints. O `memoryId` não oferece suporte a idempotência; apenas o `customId` oferece.
</Note>

<div id="single-delete">
  ## Exclusão individual
</div>

Exclua memórias individuais pelo seu id. Esta é uma exclusão definitiva (hard delete), sem possibilidade de recuperação.

<CodeGroup>

```typescript Typescript
// Hard delete — remove a memory permanentemente
await client.memories.delete('memory_id_123');
console.log('Memory deleted successfully');
```

```python Python
# Hard delete — remove a memory permanentemente
client.memories.delete('memory_id_123')
print('Memory deleted successfully')

# Tratamento de erros para exclusão individual
try:
    client.memories.delete('memory_id_123')
    print('Delete successful')
except NotFoundError:
    print('Memory not found or already deleted')
except AuthenticationError:
    print('Authentication failed')
except Exception as e:
    print(f'Delete failed: {e}')
```

```bash cURL
curl -X DELETE "https://api.supermemory.ai/v3/documents/memory_id_123" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"

# Resposta: 204 No Content (sucesso)
# Resposta: 404 Not Found (a memory não existe)
```

</CodeGroup>

<div id="bulk-delete-by-ids">
  ## Exclusão em lote por IDs
</div>

Exclua várias memórias de uma só vez fornecendo um array de IDs de memory. Máximo de 100 IDs por solicitação.

<CodeGroup>

```typescript Typescript
// Exclusão em lote por IDs de memory
const result = await client.memories.bulkDelete({
  ids: [
    'memory_id_1',
    'memory_id_2',
    'memory_id_3',
    'non_existent_id'  // Isso será informado em errors
  ]
});

console.log('Resultado da exclusão em lote:', result);
// Output: {
//   success: true,
//   deletedCount: 3,
//   errors: [
//     { id: "non_existent_id", error: "Memory not found" }
//   ]
// }
```

```python Python
# Exclusão em lote por IDs de memory
result = client.memories.bulk_delete(

        'memory_id_1',
        'memory_id_2',
        'memory_id_3',
        'non_existent_id'  # Isso será informado em errors

print(f'Resultado da exclusão em lote: {result}')

#   'success': True,
```

<div id="soft-delete-implementation">
  ### Implementação de soft delete
</div>

Para aplicações que exigem trilhas de auditoria ou mecanismos de recuperação, implemente padrões de soft delete usando metadata:

<CodeGroup>

```typescript Typescript
// Padrão de soft delete usando metadata
await client.memories.update('memory_id', {
  metadata: {
    deleted: true,
    deletedAt: new Date().toISOString(),
    deletedBy: 'user_123'
  }
});

// Excluir memórias deletadas das pesquisas
const activeMemories = await client.memories.list({
  filters: JSON.stringify({
    AND: [
      { key: "deleted", value: "true", negate: true }
    ]
  })
});

console.log('Memórias ativas:', activeMemories.results.length);
```

```python Python
from datetime import datetime
import json

# Padrão de soft delete usando metadata
client.memories.update('memory_id', {
    'metadata': {
        'deleted': True,
        'deletedAt': datetime.now().isoformat(),
        'deletedBy': 'user_123'
    }
})

# Excluir memórias deletadas dos resultados
active_memories = client.memories.list(
    filters=json.dumps({
        "AND": [
            {"key": "deleted", "value": "true", "negate": True}
        ]
    })
)

print(f'Memórias ativas: {len(active_memories.results)}')
```

<div id="batch-processing-for-large-operations">
  ### Processamento em lote para operações de grande porte
</div>

<CodeGroup>

```typescript Typescript
// Excluir em lote grandes quantidades de memórias com segurança
async function batchDeleteMemories(memoryIds: string[], batchSize = 100) {
  const results = [];

  for (let i = 0; i < memoryIds.length; i += batchSize) {
    const batch = memoryIds.slice(i, i + batchSize);

    console.log(`Processando lote ${Math.floor(i/batchSize) + 1} de ${Math.ceil(memoryIds.length/batchSize)}`);

    try {
      const result = await client.memories.bulkDelete({ ids: batch });
      results.push(result);

      // Pequeno intervalo entre lotes para evitar limitação de taxa
      if (i + batchSize < memoryIds.length) {
        await new Promise(resolve => setTimeout(resolve, 1000));
      }
    } catch (error) {
      console.error(`Falha no lote ${Math.floor(i/batchSize) + 1}:`, error);
      results.push({ success: false, error: error.message, batch });
    }
  }

  // Agregar resultados
  const totalDeleted = results
    .filter(r => r.success)
    .reduce((sum, r) => sum + (r.deletedCount || 0), 0);

  console.log(`Total excluído: ${totalDeleted} de ${memoryIds.length}`);
  return { totalDeleted, results };
}
```

```python Python
import time
import math

def batch_delete_memories(memory_ids, batch_size=100):
    """Excluir em lote grandes quantidades de memórias com segurança"""
    results = []

    for i in range(0, len(memory_ids), batch_size):
        batch = memory_ids[i:i + batch_size]
        batch_num = i // batch_size + 1
        total_batches = math.ceil(len(memory_ids) / batch_size)

        print(f'Processando lote {batch_num} de {total_batches}')

        try:
            result = client.memories.bulk_delete(ids=batch)
            results.append(result)

            # Pequeno intervalo entre lotes para evitar limitação de taxa
            if i + batch_size < len(memory_ids):
                time.sleep(1)
        except Exception as error:
            print(f'Falha no lote {batch_num}: {error}')
            results.append({'success': False, 'error': str(error), 'batch': batch})

    # Agregar resultados
    total_deleted = sum(
        r.get('deletedCount', 0) for r in results if r.get('success')
    )

    print(f'Total excluído: {total_deleted} de {len(memory_ids)}')
    return {'totalDeleted': total_deleted, 'results': results}
```

```bash cURL
# Exemplo de script de processamento em lote
#!/bin/bash

MEMORY_IDS=("id1" "id2" "id3")  # Sua lista de IDs de memória
BATCH_SIZE=100
TOTAL_DELETED=0

# Processar em lotes
for ((i=0; i<${#MEMORY_IDS[@]}; i+=BATCH_SIZE)); do
    batch=("${MEMORY_IDS[@]:i:BATCH_SIZE}")
    batch_json=$(printf '%s\n' "${batch[@]}" | jq -R . | jq -s .)

    echo "Processando lote $((i/BATCH_SIZE + 1))"

    response=$(curl -s -X DELETE \
      "https://api.supermemory.ai/v3/documents/bulk" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d "{\"ids\": $batch_json}")

    deleted_count=$(echo "$response" | jq -r '.deletedCount // 0')
    TOTAL_DELETED=$((TOTAL_DELETED + deleted_count))

    echo "Lote excluído: $deleted_count memórias"
    sleep 1  # Proteção contra limitação de taxa
done

echo "Total excluído: $TOTAL_DELETED memórias"
```

<div id="best-practices">
  ## Boas práticas
</div>

<div id="update-operations">
  ### Operações de atualização
</div>

1. **Use customId para atualizações idempotentes** - Evita memórias duplicadas e permite novas tentativas com segurança
2. **Monitore o status de processamento** - As atualizações acionam todo o pipeline de reprocessamento
3. **Trate a metadata com cuidado** - As atualizações substituem as chaves de metadata especificadas
4. **Implemente um tratamento de erros adequado** - A memory pode ser excluída entre operações

<div id="delete-operations">
  ### Operações de exclusão
</div>

1. **Hard delete é permanente** - Não há mecanismo de recuperação
2. **Use operações em lote de forma eficiente** - Máximo de 100 IDs por solicitação de exclusão em lote
3. **Considere padrões de soft delete** - Use flags em metadata para exclusões recuperáveis
4. **Faça batching de operações grandes** - Evite limites de taxa com um batching adequado
5. **Limpe o estado do aplicativo** - Atualize sua UI/cache após as exclusões
