<div id="direct-updates">
  ## 直接更新
</div>

変更したいメモリーが特定できている場合は、そのIDを指定して既存のメモリーを更新します。変更はフルパイプラインでの再処理をトリガーします。

<CodeGroup>

```typescript Typescript
import Supermemory from 'supermemory';

const client = new Supermemory({
  apiKey: process.env.SUPERMEMORY_API_KEY!
});

// メモリーIDで更新
const updated = await client.memories.update('memory_id_123', {
  content: 'Updated content here',
  metadata: { version: 2, updated: true }
});

console.log(updated.status); // 再処理のための "queued"
console.log(updated.id); // "memory_id_123"
```

```python Python
from supermemory import Supermemory
import os

client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

# メモリーIDで更新
updated = client.memories.update(
    'memory_id_123',
    content='Updated content here',
    metadata={'version': 2, 'updated': True}
)

print(f"Status: {updated.status}")  # 再処理のための "queued"
print(f"ID: {updated.id}")  # "memory_id_123"
```

```bash cURL
curl -X PATCH "https://api.supermemory.ai/v3/documents/memory_id_123" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Updated content here",
    "metadata": {"version": 2, "updated": true}
  }'
```

</CodeGroup>

<div id="upserts-using-customid">
  ## customId を使ったアップサート
</div>

`add()` に同じ `customId` を指定すると、重複を作らず既存のメモリーを更新します。こうした冪等な処理には `customId` を使用します。

<CodeGroup>

```typescript Typescript
import Supermemory from 'supermemory';

const client = new Supermemory({
  apiKey: process.env.SUPERMEMORY_API_KEY!
});

const customId = 'user-note-001';

// First call creates memory
const created = await client.memories.add({
  content: 'Initial content',
  customId: customId,
  metadata: { version: 1 }
});

console.log('Created memory:', created.id);

// Second call with same customId updates existing
const updated = await client.memories.add({
  content: 'Updated content',
  customId: customId,         // Same customId = upsert
  metadata: { version: 2 }
});
```

```python Python
from supermemory import Supermemory
import os

client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

custom_id = 'user-note-001'

# First call creates memory
created = client.memories.add(
    content='Initial content',
    custom_id=custom_id,
    metadata={'version': 1}
)

print(f'Created memory: {created.id}')

# Second call with same customId updates existing
updated = client.memories.add(
    content='Updated content',
    custom_id=custom_id,     # Same customId = upsert
    metadata={'version': 2}
)

print(f'Updated memory: {updated.id}')
print(f'Same memory? {created.id == updated.id}')  # True
```

```bash cURL
# First call - creates memory
curl -X POST "https://api.supermemory.ai/v3/documents" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Initial content",
    "customId": "user-note-001",
    "metadata": {"version": 1}
  }'

# Response: {"id": "mem_abc123", "status": "queued", "customId": "user-note-001"}

# Second call - updates existing (same customId)
curl -X POST "https://api.supermemory.ai/v3/documents" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
  -H "Content-Type: "application/json" \
  -d '{
    "content": "Updated content",
    "customId": "user-note-001",
    "metadata": {"version": 2}
  }'

# Response: {"id": "mem_abc123", "status": "queued", "customId": "user-note-001"}
# Note: Same ID returned - memory was updated, not created
```

</CodeGroup>

<Note>
`customId` はすべてのエンドポイントで冪等性を提供します。`memoryId` は冪等性をサポートせず、対応するのは `customId` のみです。
</Note>

<div id="single-delete">
  ## 単体削除
</div>

id を指定してメモリーを個別に削除します。これは復元できない恒久的なハード削除です。

<CodeGroup>

```typescript Typescript
// ハード削除 — メモリーを完全に削除します
await client.memories.delete('memory_id_123');
console.log('メモリーを削除しました');
```

```python Python
# ハード削除 — メモリーを完全に削除します
client.memories.delete('memory_id_123')
print('メモリーを削除しました')

# 単体削除のエラーハンドリング
try:
    client.memories.delete('memory_id_123')
    print('削除に成功しました')
except NotFoundError:
    print('メモリーが見つからない、またはすでに削除されています')
except AuthenticationError:
    print('認証に失敗しました')
except Exception as e:
    print(f'削除に失敗しました: {e}')
```

```bash cURL
curl -X DELETE "https://api.supermemory.ai/v3/documents/memory_id_123" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"

# 応答: 204 No Content（成功）
# 応答: 404 Not Found（メモリーが存在しません）
```

</CodeGroup>

<div id="bulk-delete-by-ids">
  ## ID による一括削除
</div>

メモリーの ID 配列を指定して、複数のメモリーをまとめて削除します。1 回のリクエストで指定できる ID は最大 100 個です。

<CodeGroup>

```typescript Typescript
// メモリーの ID による一括削除
const result = await client.memories.bulkDelete({
  ids: [
    'memory_id_1',
    'memory_id_2',
    'memory_id_3',
    'non_existent_id'  // これは errors に報告されます
  ]
});

console.log('一括削除の結果:', result);
// 出力: {
//   success: true,
//   deletedCount: 3,
//   errors: [
//     { id: "non_existent_id", error: "Memory not found" }
//   ]
// }
```

```python Python
# メモリーの ID による一括削除
result = client.memories.bulk_delete(
```

<div id="soft-delete-implementation">
  ### ソフトデリートの実装
</div>

監査ログや復旧メカニズムが必要なアプリケーションでは、メタデータを使ったソフトデリートパターンを実装します:

<CodeGroup>

```typescript Typescript
// メタデータを用いたソフトデリートパターン
await client.memories.update('memory_id', {
  metadata: {
    deleted: true,
    deletedAt: new Date().toISOString(),
    deletedBy: 'user_123'
  }
});

// 検索時に削除済みメモリーを除外
const activeMemories = await client.memories.list({
  filters: JSON.stringify({
    AND: [
      { key: "deleted", value: "true", negate: true }
    ]
  })
});

console.log('Active memories:', activeMemories.results.length);
```

```python Python
from datetime import datetime
import json

# メタデータを用いたソフトデリートパターン
client.memories.update('memory_id', {
    'metadata': {
        'deleted': True,
        'deletedAt': datetime.now().isoformat(),
        'deletedBy': 'user_123'
    }
})

# 削除済みメモリーを除外
active_memories = client.memories.list(
    filters=json.dumps({
        "AND": [
            {"key": "deleted", "value": "true", "negate": True}
        ]
    })
)

print(f'Active memories: {len(active_memories.results)}')
```

<div id="batch-processing-for-large-operations">
  ### 大規模処理のバッチ実行
</div>

<CodeGroup>

```typescript Typescript
// 多数のメモリーを安全にバッチ削除する
async function batchDeleteMemories(memoryIds: string[], batchSize = 100) {
  const results = [];

  for (let i = 0; i < memoryIds.length; i += batchSize) {
    const batch = memoryIds.slice(i, i + batchSize);

    console.log(`バッチ処理 ${Math.floor(i/batchSize) + 1} / ${Math.ceil(memoryIds.length/batchSize)} を実行中`);

    try {
      const result = await client.memories.bulkDelete({ ids: batch });
      results.push(result);

      // レート制限を避けるため、バッチ間で短時間待機
      if (i + batchSize < memoryIds.length) {
        await new Promise(resolve => setTimeout(resolve, 1000));
      }
    } catch (error) {
      console.error(`バッチ ${Math.floor(i/batchSize) + 1} の処理に失敗:`, error);
      results.push({ success: false, error: error.message, batch });
    }
  }

  // 結果の集計
  const totalDeleted = results
    .filter(r => r.success)
    .reduce((sum, r) => sum + (r.deletedCount || 0), 0);

  console.log(`削除合計: ${totalDeleted} / ${memoryIds.length}`);
  return { totalDeleted, results };
}
```

```python Python
import time
import math

def batch_delete_memories(memory_ids, batch_size=100):
    """多数のメモリーを安全にバッチ削除する"""
    results = []

    for i in range(0, len(memory_ids), batch_size):
        batch = memory_ids[i:i + batch_size]
        batch_num = i // batch_size + 1
        total_batches = math.ceil(len(memory_ids) / batch_size)

        print(f'バッチ処理 {batch_num} / {total_batches} を実行中')

        try:
            result = client.memories.bulk_delete(ids=batch)
            results.append(result)

            # レート制限を避けるため、バッチ間で短時間待機
            if i + batch_size < len(memory_ids):
                time.sleep(1)
        except Exception as error:
            print(f'バッチ {batch_num} の処理に失敗: {error}')
            results.append({'success': False, 'error': str(error), 'batch': batch})

    # 結果の集計
    total_deleted = sum(
        r.get('deletedCount', 0) for r in results if r.get('success')
    )

    print(f'削除合計: {total_deleted} / {len(memory_ids)}')
    return {'totalDeleted': total_deleted, 'results': results}
```

```bash cURL
# バッチ処理スクリプト例
#!/bin/bash

MEMORY_IDS=("id1" "id2" "id3")  # メモリーIDの配列
BATCH_SIZE=100
TOTAL_DELETED=0

# バッチで処理
for ((i=0; i<${#MEMORY_IDS[@]}; i+=BATCH_SIZE)); do
    batch=("${MEMORY_IDS[@]:i:BATCH_SIZE}")
    batch_json=$(printf '%s\n' "${batch[@]}" | jq -R . | jq -s .)

    echo "バッチ $((i/BATCH_SIZE + 1)) を処理中"

    response=$(curl -s -X DELETE \
      "https://api.supermemory.ai/v3/documents/bulk" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d "{\"ids\": $batch_json}")

    deleted_count=$(echo "$response" | jq -r '.deletedCount // 0')
    TOTAL_DELETED=$((TOTAL_DELETED + deleted_count))

    echo "このバッチで削除: $deleted_count 件のメモリー"
    sleep 1  # レート制限対策
done

echo "削除合計: $TOTAL_DELETED 件のメモリー"
```

<div id="best-practices">
  ## ベストプラクティス
</div>

<div id="update-operations">
  ### 更新オペレーション
</div>

1. **冪等な更新には customId を使用** - 保存メモリの重複を防ぎ、安全にリトライできる
2. **処理 status を監視** - 更新は再処理パイプライン全体をトリガーする
3. **メタデータの取り扱いに注意** - 更新すると指定したメタデータのキーが置き換えられる
4. **適切なエラーハンドリングを実装** - オペレーションの合間にメモリーが削除されている可能性がある

<div id="delete-operations">
  ### 削除オペレーション
</div>

1. **ハード削除は永久的** - 復元手段はありません
2. **バルク操作を効率的に活用** - バルク削除リクエストあたりIDは最大100件
3. **ソフト削除のパターンを検討** - 復元可能な削除にはメタデータのフラグを使用
4. **大規模処理はバッチ化** - 適切なバッチングでレート制限を回避
5. **アプリケーションの状態をクリーンアップ** - 削除後に UI／キャッシュを更新
