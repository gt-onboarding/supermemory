APIの更新、新規エンドポイント、SDKのリリース、開発者向けの新機能。

<div id="september-17-2025">
  ## 2025年9月17日
</div>

- **Forgotten Memories 検索:** v4 検索 API に新たに追加された `include.forgottenMemories` パラメータにより、明示的に忘れられた、または期限切れになった保存メモリも検索できるようになりました。`true` に設定すると、検索結果に忘却済みの保存メモリを含められ、以前にアーカイブされた情報の復元に役立ちます。

<div id="september-14-2025">
  ## 2025年9月14日
</div>

- **Delete API の強化:** `DELETE /v3/documents/:id` エンドポイントが、柔軟なドキュメント削除のために内部ドキュメントIDと customId の両方に対応しました。開発者は作成時に指定したのと同じ customId を使って documents を削除できるようになり、他のエンドポイントとの API 一貫性が向上します。
- **API 用語の明確化:** 開発者の理解を高めるため、API の用語を「memories」から「documents」へと整理しました。新しい `/v3/documents/*` エンドポイントは、より直感的な命名を提供しつつ、`/v3/memories/*` からの自動リダイレクトにより完全な後方互換性を維持します。既存の統合での対応は不要です。

<div id="september-13-2025">
  ## 2025年9月13日
</div>

- **Documentation v2.0:** TypeScript、Python、cURL向けの包括的なAPIリファレンス、クックブック、プロダクションレディなサンプルを備えた全面改訂
- **AI SDK Integration:** Vercel AI SDKとネイティブに統合し、Memory ToolsとInfinite Chat（インフィニット・チャット）に対応する新しい`@supermemory/tools/ai-sdk`パッケージ
- **Bulk Delete Endpoint:** メモリーを効率的に管理できる新しい`DELETE /v3/documents/bulk`エンドポイント

<div id="september-5-2025">
  ## 2025年9月5日
</div>

- **メモリー検索エンドポイント:** 会話型AIとメモリー取得に最適化された新しい `/v4/search` エンドポイント（ドキュメント検索とは対照的）
- **高度なメモリー管理:** フィルタリングとバッチ処理を強化した更新・削除操作

<div id="august-30-2025">
  ## 2025年8月30日
</div>

- **MCP (Model Context Protocol) サーバー:** AI モデル統合向けの自動検出とプロジェクト全体のサポートを備えた supermemory MCP サーバーをリリース
- **フィルタリング API の強化:** SQL ベースのフィルタリングに array_contains、数値演算子、複雑な AND/OR ロジックへの対応を追加

<div id="august-15-2025">
  ## 2025年8月15日
</div>

- **Memory Router Proxy:** LLMリクエスト向けのプロキシ機能を強化し、自動コンテキスト管理とトークン最適化に対応
- **Search Algorithm Updates:** 結果の品質向上に向けた、設定可能な類似度のthreshold、リエランキング、クエリ リライト

<div id="april-30-2025">
  ## 2025年4月30日
</div>

- **包括的なAPIドキュメント:** パラメーターの詳細解説とレスポンススキーマを備えた新しいインタラクティブなAPIリファレンス
- **コンテナタグシステム:** メモリーの分離やユーザー単位のコンテンツ管理を強化するための拡張されたグルーピング機能
- **コンテンツタイプの自動検出:** URLの拡張子に依存せず、PDF・画像・動画・Webコンテンツを自動的に処理

<div id="april-28-2025">
  ## 2025年4月28日
</div>

- **Google Drive Connector API:** Google Driveのプログラム連携とファイル同期向けの新規エンドポイント

<div id="april-25-2025">
  ## 2025年4月25日
</div>

- **検索しきい値の制御:** 検索感度を細かく調整できる新しい `documentThreshold` と `chunkThreshold` パラメータ
- **ドキュメント単位の検索:** 特定の大規模な documents 内を検索するための新しい `docId` パラメータ
- **チャンク制御の強化:** 精密な結果フィルタリングのための `onlyMatchingChunks` パラメータ

<div id="april-24-2025">
  ## 2025年4月24日
</div>

- **クエリ リライト API:** 検索結果の精度向上に向けた自動クエリ拡張と意図のマッチング
- **検索コンテキストのオプション:** 包括的なドキュメント取得のための `includeFullDocs` と `includeSummary` パラメータを追加

<div id="april-18-2025">
  ## 2025年4月18日
</div>

- **コンテンツ処理の強化:** 画像・動画・PDFのURLを直接処理できるようにした、改良版のインジェストパイプライン
- **Webインジェストの安定化:** コンテンツ抽出を改善し、ウェブサイトのURL処理をより高信頼に

<div id="april-14-2025">
  ## 2025年4月14日
</div>

- **Team API エンドポイント:** チーム管理と権限のコントロール向けの新規エンドポイント
- **Analytics API の強化:** 詳細な利用メトリクスとパフォーマンスデータにより可観測性が向上

<div id="february-1-2025">
  ## 2025年2月1日
</div>

- **マルチスペース検索:** 配列パラメータに対応し、複数のコンテナタグを同時に横断検索
- **API バージョニング:** 改良されたバージョニング戦略に基づく `/v1` エンドポイントへの移行
- **インタラクティブ API Playground:** すべてのエンドポイントを対象とした、ライブ例付きの新しいテスト用インターフェース
