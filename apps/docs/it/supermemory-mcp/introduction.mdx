supermemory MCP è un componente leggero rivolto agli utenti finali che ti consente di portare con te le tue memorie su qualsiasi piattaforma AI. Funziona come un livello di memoria universale che permette ai Large Language Models (LLM) di mantenere un contesto e memorie persistenti tra applicazioni e sessioni diverse, risolvendo la limitazione fondamentale degli assistenti AI che dimenticano tutto tra una conversazione e l'altra.

<div id="what-supermemory-mcp-is-and-how-it-works">
  ## Che cos'è supermemory MCP e come funziona
</div>

**supermemory MCP** è un sistema di memory open-source e universale che connette applicazioni di AI tra loro tramite il Model Context Protocol (MCP). Funziona come un **server MCP meta** che crea istanze di server dedicate per ogni utente, garantendo una persistenza della memory continua su qualsiasi client LLM compatibile con MCP, inclusi ChatGPT, Claude, Windsurf, Cursor e VS Code.

Il sistema opera come server MCP che comunica con client compatibili, archiviando e recuperando informazioni contestuali attraverso la solida infrastruttura della Memory API di supermemory. Quando gli utenti interagiscono con qualsiasi applicazione di AI connessa, il sistema acquisisce automaticamente le informazioni pertinenti e le rende disponibili a tutte le altre piattaforme collegate tramite **ricerca semantica e recupero intelligente**.

È anche un ottimo esempio di come utilizzare supermemory per creare applicazioni consumer.

<div id="key-consumer-benefits">
  ### Vantaggi principali per gli utenti
</div>

- **Nessun accesso con credenziali**: Accesso tramite URL univoci e privati che fungono sia da identificatore sia da chiave API
- **Completamente gratuito**: Servizio ospitato, pienamente funzionante, disponibile senza costi
- **Configurazione con un solo comando**: Installazione unica da CLI: `npx install-mcp [URL] --client [CLIENT]`
- **Compatibilità universale**: Funziona senza problemi con più client e piattaforme LLM

<div id="core-workflow-process">
  ### Flusso di lavoro principale
</div>

1. L’utente interagisce con qualsiasi client AI compatibile con MCP
2. Il client si connette al server MCP di Supermemory tramite un URL univoco e privato
3. Durante le conversazioni, le informazioni pertinenti vengono archiviate usando l’azione `addToSupermemory`
4. Quando serve contesto, l’azione `searchSupermemory` recupera le memories pertinenti
5. L’assistente AI accede a questo contesto persistente indipendentemente dalla piattaforma utilizzata

<div id="building-applications-with-supermemory">
  ## Creare applicazioni con supermemory
</div>

supermemory MCP mostra come sviluppare applicazioni consumer con supermemory grazie alla sua **architettura API-first**. Il sistema si comporta come un “cliente” dell’infrastruttura sottostante della Supermemory Memory API, illustrando come gli sviluppatori possano sfruttare la piattaforma per creare applicazioni abilitate alla memory.

<div id="supported-platforms">
  ### Piattaforme supportate
</div>

- **Claude Desktop**: Supporto diretto per connessioni SSE
- **Cursor IDE**: Configurazione globale del server MCP tramite `~/.cursor/mcp.json`
- **Windsurf**: Integrazione fluida per lo sviluppo potenziato dall'AI
- **VS Code**: Compatibile con varie estensioni di AI per il coding
- **Cline/Roo-Cline**: Pieno supporto per il protocollo MCP
- **Qualsiasi applicazione compatibile con MCP**: Compatibilità universale grazie all'implementazione del protocollo standard

<div id="security-and-privacy-considerations">
  ## Considerazioni su sicurezza e privacy
</div>

<div id="security-model">
  ### Modello di sicurezza
</div>

- **Autenticazione basata su URL**: URL casuali fungono da chiavi di accesso, eliminando la complessità dell’autenticazione tradizionale
- **Isolamento delle sessioni**: Separazione completa dei dati degli utenti tramite percorsi URL univoci
- **Opzione di self-hosting**: Pieno controllo su dati e infrastruttura per gli utenti attenti alla privacy
- **Nessun overhead di autenticazione**: Accesso semplificato senza sistemi tradizionali di nome utente/password

<div id="privacy-features">
  ### Funzionalità per la privacy
</div>

- **Isolamento dei dati**: memories degli utenti completamente separati tramite URL univoci
- **Opzione di controllo locale**: possibilità di self-hosting per aziende o casi d’uso sensibili
- **Infrastruttura sicura**: basata sulla piattaforma di sicurezza enterprise di Cloudflare
