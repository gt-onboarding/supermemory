Supermemory supporta il filtraggio dei risultati di ricerca usando tag di container, condizioni sui metadata e tecniche avanzate di filtraggio per gli endpoint `/v3/search` e `/v4/search`.

<div id="container-tags">
  ## Tag di contenitore
</div>

I tag di contenitore raggruppano le memories per utente, progetto o organizzazione. Sono il modo principale per isolare i risultati di ricerca.

**Importante**: i tag di contenitore usano il **matching esatto degli array**. Un documento con `["technology", "quantum-computing"]` NON corrisponderà a una ricerca per `["technology"]`. Gli array devono essere identici.

<div id="document-search-v3search">
  ### Ricerca documenti (v3/search)
</div>

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    // Tag contenitore singolo
    const results = await client.search.documents({
      q: "machine learning",
      containerTags: ["user_123"],
      limit: 10
    });

    // Più tag contenitore
    const results2 = await client.search.documents({
      q: "project status",
      containerTags: ["project_ai", "team_research"],
      limit: 10
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    # Tag contenitore singolo
    results = client.search.documents(
        q="machine learning",
        container_tags=["user_123"],
        limit=10
    )

    # Più tag contenitore
    results2 = client.search.documents(
        q="project status",
        container_tags=["project_ai", "team_research"],
        limit=10
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    # Tag contenitore singolo
    curl -X POST "https://api.supermemory.ai/v3/search" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "q": "machine learning",
        "containerTags": ["user_123"],
        "limit": 10
      }'
    ```
  </Tab>
</Tabs>

<Note>
**Differenze dei container tag**:
- `/v3/search` usa `containerTags` (array al plurale) con corrispondenza esatta dell’array
- `/v4/search` usa `containerTag` (stringa al singolare) per il filtraggio con un singolo tag
- Corrispondenza esatta significa che `["user", "project"]` ≠ `["user"]`
</Note>

<div id="array-contains-filtering">

</div>

<div id="filter-best-practices">
  ## Best practice per i filtri
</div>

<Callout type="info">
**Suggerimenti per le prestazioni**:
- Usa i tag contenitore come filtro principale (il più veloce)
- Combinali con semplici filtri sulla metadata per maggiore precisione
- Evita condizioni OR annidate complesse per prestazioni migliori
- Usa operatori numerici solo quando necessario
</Callout>

<Warning>
**Struttura dei filtri**: tutte le condizioni devono essere racchiuse in array `AND` o `OR`. Anche una singola condizione richiede l’array wrapper: `{"AND": [{"key": "category", "value": "ai"}]}`.
</Warning>

<Note>
**Operatori supportati**:
- **Stringa**: solo corrispondenza esatta
- **Numerico**: `<=`, `>=`, `<`, `>`, `=`
- **Array**: `array_contains` per verificare se l’array contiene un valore
- **Negazione**: funziona con tutti i tipi di filtro
</Note>
