API 更新、新增端点、SDK 发布，以及面向开发者的功能。

<div id="september-17-2025">
  ## 2025年9月17日
</div>

- **遗忘记忆搜索：** 在 v4 搜索 API 中新增 `include.forgottenMemories` 参数，允许搜索那些被明确标记为遗忘或已过期的记忆存储。将其设为 `true` 可在搜索结果中包含这类记忆存储，有助于找回先前归档的信息。

<div id="september-14-2025">
  ## 2025 年 9 月 14 日
</div>

- **增强的 Delete API：** `DELETE /v3/documents/:id` 端点现已同时支持内部文档 id 和 customId，便于灵活删除文档。开发者现在可以使用在创建时提供的同一 customId 来删除文档，从而提升该 API 与其他端点的一致性。
- **API 术语澄清：** 将 API 术语从“记忆存储”调整为“Documents”，以便开发者更易理解。新的 `/v3/documents/*` 端点在通过从 `/v3/memories/*` 自动重定向实现完全向后兼容的同时，提供更直观的命名。现有集成无需任何改动。

<div id="september-13-2025">
  ## 2025年9月13日
</div>

- **Documentation v2.0：** 全面重写，提供完整的 API 参考、cookbook 操作示例，以及适用于 TypeScript、Python 和 cURL 的生产级示例
- **AI SDK 集成：** 全新 `@supermemory/tools/ai-sdk` 包，原生集成 Vercel AI SDK，支持 Memory Tools 与 Infinite Chat 功能
- **批量删除端点：** 新增 `DELETE /v3/documents/bulk` 端点，用于高效的 memory 管理

<div id="september-5-2025">
  ## 2025年9月5日
</div>

- **Memory 搜索端点：** 新增 `/v4/search` 端点，专为会话式 AI 与 memory 检索优化（区别于文档搜索）
- **高级 Memory 管理：** 加强更新/删除操作，提供更完善的筛选与批处理能力

<div id="august-30-2025">
  ## 2025年8月30日
</div>

- **MCP（Model Context Protocol）服务器：** 发布 supermemory MCP 服务器，用于 AI 模型集成，具备完整项目支持与自动检测
- **增强的过滤 API：** 改进基于 SQL 的过滤功能，支持 array_contains、数值型运算符，以及复杂的 AND/OR 逻辑

<div id="august-15-2025">
  ## 2025年8月15日
</div>

- **Memory Router 代理：** 增强对 LLM 请求的代理能力，自动管理上下文并优化 Token 使用
- **搜索算法更新：** 可配置的相似性阈值、重排与查询重写，以提升结果质量

<div id="april-30-2025">
  ## 2025年4月30日
</div>

- **全面的 API 文档：** 全新的交互式 API 参考，附详细参数说明和响应架构
- **容器标签系统：** 更强的组织分组能力，实现更好的 memory 隔离与按用户范围的内容管理
- **自动内容类型检测：** 即使不含 URL 扩展名也可自动处理 PDF、图像、视频和网页内容

<div id="april-28-2025">
  ## 2025年4月28日
</div>

- **Google Drive Connector API：** 新增用于以编程方式集成 Google Drive 并同步文件的端点

<div id="april-25-2025">
  ## 2025 年 4 月 25 日
</div>

- **搜索阈值控制：** 新增 `documentThreshold` 和 `chunkThreshold` 参数，用于精细调节搜索灵敏度
- **特定文档搜索：** 新增 `docId` 参数，可在指定的大型 Documents 中进行搜索
- **增强的 chunk 控制：** 新增 `onlyMatchingChunks` 参数，用于精确过滤结果

<div id="april-24-2025">
  ## 2025年4月24日
</div>

- **查询重写 API：** 自动进行查询扩展与意图匹配，提升搜索效果
- **搜索上下文选项：** 新增 `includeFullDocs` 和 `includeSummary` 参数，支持更全面的文档获取

<div id="april-18-2025">
  ## 2025年4月18日
</div>

- **强化的内容处理：** 改进的摄取流水线，支持对图像、视频和 PDF 的 URL 直接处理
- **更稳定的网页摄取：** 网站 URL 处理更可靠，内容提取效果更佳

<div id="april-14-2025">
  ## 2025年4月14日
</div>

- **Team API Endpoints：** 新增团队管理与权限控制相关端点
- **Enhanced Analytics API：** 提升可观测性，提供更详细的使用指标与性能数据

<div id="february-1-2025">
  ## 2025年2月1日
</div>

- **多空间搜索：** 支持数组参数，可同时在多个容器标签中搜索
- **API 版本控制：** 迁移至 `/v1` 端点，并优化版本策略
- **交互式 API Playground：** 面向所有端点的全新测试界面，提供实时示例
