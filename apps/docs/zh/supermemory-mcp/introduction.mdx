supermemory MCP 是一个轻量级的**面向消费者**组件，允许你在任意 AI 平台上随身携带你的记忆。它作为通用的 memory 层，使 LLM（大型语言模型）能够在不同应用与会话中保持持久的上下文与记忆，从而解决 AI 助手在对话之间“遗忘”的根本性限制。

<div id="what-supermemory-mcp-is-and-how-it-works">
  ## supermemory MCP 是什么，它如何工作
</div>

**supermemory MCP** 是一个开源的通用 memory 系统，通过 Model Context Protocol（MCP）将彼此孤立的 AI 应用连接起来。它作为**元 MCP 服务器**运行，为每位用户创建独立的服务器实例，并在包括 ChatGPT、Claude、Windsurf、Cursor、VS Code 在内的任何兼容 MCP 的 LLM（大型语言模型）客户端之间提供无缝的 memory 持久化。

该系统以 MCP 服务器形式运行，与兼容 MCP 的客户端通信；依托稳健的 Supermemory API 基础设施存储和检索上下文信息。当用户与任一已连接的 AI 应用交互时，系统会自动捕获相关信息，并通过**语义搜索与智能检索**将其同步供其他已连接的平台使用。

它也是一个很好的示例，展示了如何使用 supermemory 构建面向消费者的应用。

<div id="key-consumer-benefits">
  ### 核心用户收益
</div>

- **无需登录**：通过唯一且私密的 URL 授权访问，同时兼作标识符和 API 密钥
- **完全免费**：提供功能完备的托管服务，零成本使用
- **一条命令完成设置**：单条 CLI 安装命令：`npx install-mcp [URL] --client [CLIENT]`
- **广泛兼容**：可在多个 LLM 客户端与平台上无缝运行

<div id="core-workflow-process">
  ### 核心工作流程
</div>

1. 用户与任意兼容 MCP 的 AI 客户端进行交互
2. 客户端通过唯一的私有 url 连接到 supermemory MCP 服务器
3. 在对话过程中，使用 `addToSupermemory` 动作存储相关信息
4. 当需要上下文时，使用 `searchSupermemory` 动作检索相关记忆存储
5. 无论使用哪个平台，AI 助手都可以访问这些持久化的上下文

<div id="building-applications-with-supermemory">
  ## 使用 supermemory 构建应用
</div>

supermemory MCP 通过其**API 优先架构（API-first architecture）**展示了如何使用 supermemory 构建消费者应用。该系统作为底层 supermemory API 基础设施的“客户端”，展示了开发者如何借助该平台创建具备 memory 能力的自有应用。

<div id="supported-platforms">
  ### 支持的平台
</div>

- **Claude Desktop**：原生支持 SSE 连接
- **Cursor IDE**：通过 `~/.cursor/mcp.json` 配置全局 MCP 服务器
- **Windsurf**：与 AI 驱动开发无缝集成
- **VS Code**：兼容多种 AI 编程扩展
- **Cline/Roo-Cline**：完整支持 MCP 协议
- **任何 MCP 兼容应用**：通过标准协议实现实现通用兼容性

<div id="security-and-privacy-considerations">
  ## 安全与隐私注意事项
</div>

<div id="security-model">
  ### 安全模型
</div>

- **基于 URL 的身份验证**：随机 URL 作为访问密钥，省去传统登录的复杂性
- **会话隔离**：通过唯一的 URL 路径实现用户数据的彻底隔离
- **自托管选项**：为重视隐私的用户提供对数据与基础设施的完全掌控
- **无身份验证负担**：无需传统的用户名/密码体系即可简化访问

<div id="privacy-features">
  ### 隐私功能
</div>

- **数据隔离**：用户记忆存储通过唯一 URL 完全隔离
- **本地控制选项**：支持自托管，适用于企业或敏感场景
- **安全基础设施**：构建于 Cloudflare 的企业级安全平台之上
