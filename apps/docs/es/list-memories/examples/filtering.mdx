Filtra memories usando etiquetas de contenedor y metadata. El sistema de filtrado utiliza construcción de consultas SQL, por lo que debes estructurar tus filters como consultas de bases de datos.

<div id="filter-by-container-tags">
  ## Filtrar por etiquetas de contenedor
</div>

Las etiquetas de contenedor usan coincidencia exacta de arreglo: las memories deben tener exactamente las mismas etiquetas en el mismo orden.

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    // Etiqueta única: coincide con memories exactamente ["user_123"]
    const userMemories = await client.memories.list({
      containerTags: ["user_123"]
    });

    // Múltiples etiquetas: coincide con memories exactamente ["user_123", "project_ai"]
    const projectMemories = await client.memories.list({
      containerTags: ["user_123", "project_ai"]
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    # Etiqueta única
    user_memories = client.memories.list(container_tags=["user_123"])

    # Múltiples etiquetas (coincidencia exacta)
    project_memories = client.memories.list(
        container_tags=["user_123", "project_ai"]
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    # Etiqueta única
    curl -X POST "https://api.supermemory.ai/v3/documents/list" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"containerTags": ["user_123"]}'

    # Múltiples etiquetas
    curl -X POST "https://api.supermemory.ai/v3/documents/list" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"containerTags": ["user_123", "project_ai"]}'
    ```
  </Tab>
</Tabs>

<div id="metadata-filtering-with-sql-logic">
  ## Filtrado de metadata con lógica SQL
</div>

El parámetro `filters` permite filtrar por campos de metadata usando estructuras de consulta similares a SQL. Dado que construimos consultas SQL en el backend, debes estructurar tus `filters` como consultas de base de datos con lógica explícita AND/OR.

<div id="why-this-structure">
  ### ¿Por qué esta estructura?
</div>

En las bases de datos SQL, `AND` tiene mayor precedencia que `OR`. Sin un agrupamiento explícito, una consulta como:

```
category = 'programming' OR framework = 'react' AND difficulty = 'advanced'
```

Se interpreta como:

```
category = 'programming' OR (framework = 'react' AND difficulty = 'advanced')
```

La estructura JSON obliga a un agrupamiento explícito para evitar resultados inesperados.

<Info>
  **Reglas de la estructura de filtros:**

  * Envuelve siempre las condiciones en arreglos `AND` u `OR` (incluso si es una sola condición)
  * Usa `JSON.stringify()` para convertir el objeto de filtro en una cadena
  * Cada condición necesita las propiedades `key`, `value` y `negate`
  * `negate: false` para coincidencia normal, `negate: true` para exclusión
</Info>


<div id="simple-metadata-filter">
  ### Filtro sencillo de metadata
</div>

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    // Filtrar por un solo campo de metadata
    const programmingMemories = await client.memories.list({
      filters: JSON.stringify({
        AND: [
          { key: "category", value: "programming", negate: false }
        ]
      })
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import json

    # Filtrar por un solo campo de metadata
    programming_memories = client.memories.list(
        filters=json.dumps({
            "AND": [
                {"key": "category", "value": "programming", "negate": False}
            ]
        })
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/documents/list" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "filters": "{\"AND\":[{\"key\":\"category\",\"value\":\"programming\",\"negate\":false}]}"
      }'
    ```
  </Tab>
</Tabs>

<Warning>
**Errores comunes:**
- Usar objetos de condición sin envolver: `{"key": "category", "value": "programming"}`
- Olvidar JSON.stringify: pasar objetos en lugar de cadenas
- Omitir la propiedad negate: incluye siempre `"negate": false` o `"negate": true`
</Warning>

<Note>
**Etiquetas de contenedor vs. filtrado por metadata:**
- Etiquetas de contenedor: coincidencia exacta de arrays para la agrupación organizativa
- Filtros de metadata: consultas tipo SQL sobre campos de metadata personalizados con lógica compleja
- Ambos pueden combinarse para ofrecer capacidades de filtrado potentes
</Note>
