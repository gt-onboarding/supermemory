Surveillez votre usage de Supermemory grâce à des analyses détaillées des appels API, des erreurs et des indicateurs de performance.

<div id="overview">
  ## Vue d’ensemble
</div>

L’API Analytics fournit une visibilité complète sur votre utilisation de Supermemory :

- **Statistiques d’utilisation** : suivez les appels à l’API par type, les tendances horaires et la répartition par clé d’API
- **Suivi des erreurs** : identifiez les principaux types d’erreurs et leurs tendances
- **Journaux détaillés** : accédez aux journaux complets des requêtes/réponses pour le débogage
- **Indicateurs de performance** : surveillez les temps de réponse moyens et la durée de traitement

<Note>
Les données Analytics sont disponibles pour l’ensemble de votre organisation et peuvent être filtrées par période.
</Note>

<div id="usage-statistics">
  ## Statistiques d’utilisation
</div>

Accédez à des statistiques d’utilisation complètes, avec des ventilations horaires et des métriques par clé.

<div id="endpoint">
  ### Point de terminaison
</div>

`GET /v3/analytics/usage`

<div id="parameters">
  ### Paramètres
</div>

| Paramètre | Type | Description |
|-----------|------|-------------|
| `from` | string (ISO 8601) | Date/heure de début de la période |
| `to` | string (ISO 8601) | Date/heure de fin de la période |
| `period` | string | Alternative à `from` : `1h`, `24h`, `7d`, `30d` |
| `page` | integer | Numéro de page pour la pagination (valeur par défaut : 1) |
| `limit` | integer | Éléments par page (valeur par défaut : 20, max. : 100) |

<div id="example-request">
  ### Exemple de requête
</div>

<CodeGroup>

```typescript TypeScript
// Récupérer l’usage des dernières 24 heures
const usage = await fetch('https://api.supermemory.ai/v3/analytics/usage?period=24h', {
  headers: {
    'Authorization': `Bearer ${SUPERMEMORY_API_KEY}`
  }
});

const data = await usage.json();
```

```python Python
import requests
from datetime import datetime, timedelta

# Récupérer l’usage des 7 derniers jours
response = requests.get(
    'https://api.supermemory.ai/v3/analytics/usage',
    params={'period': '7d'},
    headers={'Authorization': f'Bearer {SUPERMEMORY_API_KEY}'}
)

data = response.json()
```

```bash cURL
# Récupérer l’usage pour une plage de dates spécifique
curl -X GET "https://api.supermemory.ai/v3/analytics/usage?from=2024-01-01T00:00:00Z&to=2024-01-31T23:59:59Z" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"
```

</CodeGroup>

<div id="error-monitoring">
  ## Suivi des erreurs
</div>

Suivez et analysez les erreurs pour identifier les problèmes et améliorer la fiabilité.

<div id="endpoint">
  ### Point de terminaison
</div>

`GET /v3/analytics/errors`

<div id="parameters">
  ### Paramètres
</div>

Identiques à l’endpoint d’usage — prend en charge `from`, `to`, `period`, `page` et `limit`.

<div id="example-request">
  ### Exemple de requête
</div>

<CodeGroup>

```typescript TypeScript
// Récupérer les erreurs des dernières 24 heures
const errors = await fetch('https://api.supermemory.ai/v3/analytics/errors?period=24h', {
  headers: {
    'Authorization': `Bearer ${SUPERMEMORY_API_KEY}`
  }
});

const data = await errors.json();
```

```python Python
# Surveiller les erreurs et alerter en cas de pic
response = requests.get(
    'https://api.supermemory.ai/v3/analytics/errors?period=1h',
    headers={'Authorization': f'Bearer {SUPERMEMORY_API_KEY}'}
)

data = response.json()
```

```bash cURL
# Récupérer les erreurs des 7 derniers jours
curl -X GET "https://api.supermemory.ai/v3/analytics/errors?period=7d" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"
```

</CodeGroup>

<div id="detailed-logs">
  ## Journaux détaillés
</div>

Accédez aux journaux complets des requêtes et réponses pour le débogage et l’audit.

<div id="endpoint">
  ### Endpoint
</div>

`GET /v3/analytics/logs`

<div id="parameters">
  ### Paramètres
</div>

Identiques à l’endpoint d’usage, plus des filters optionnels :

- `type` : Filtrer par type de requête (add, search, update, delete)
- `statusCode` : Filtrer par code de status HTTP
- `keyId` : Filtrer par clé d’API spécifique

<div id="example-request">
  ### Exemple de requête
</div>

<CodeGroup>

```typescript TypeScript
// Récupérer les requêtes récentes ayant échoué
const logs = await fetch('https://api.supermemory.ai/v3/analytics/logs?period=1h&statusCode=500', {
  headers: {
    'Authorization': `Bearer ${SUPERMEMORY_API_KEY}`
  }
});

const data = await logs.json();
```

```python Python
# Déboguer l’usage d’une clé d’API spécifique
response = requests.get(
    'https://api.supermemory.ai/v3/analytics/logs',
    params={
        'keyId': 'key_abc123',
        'period': '24h'
    },
    headers={'Authorization': f'Bearer {SUPERMEMORY_API_KEY}'}
)

logs = response.json()['logs']

```

```bash cURL
# Récupérer tous les logs pour le débogage
curl -X GET "https://api.supermemory.ai/v3/analytics/logs?period=1h&limit=50" \
  -H "Authorization: Bearer $SUPERMEMORY_API_KEY"
```

</CodeGroup>

<div id="rate-limits">
  ## Limites de débit
</div>

Les points de terminaison d’analytics sont soumis aux limites suivantes :

- 100 requêtes par minute et par organisation
- Plage temporelle maximale : 90 jours
- Taille de page maximale : 100 éléments

<Warning>
Les données d’analytics sont conservées pendant 90 jours. Pour une conservation plus longue, exportez et stockez les données dans vos propres systèmes.
</Warning>
